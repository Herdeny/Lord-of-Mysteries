# M2 执行计划 — 五途径序列 9-7 与 MVP 内测

> 设计基线：Project Mystery v0.6
>
> 前置门禁：M1 占卜家序列 9-7 一小时纵切通过
> 路线来源：`roadmap.json`

## 验收门槛

- 占卜家、观众、猎人、偷盗者、学徒五条途径均完成序列 9-7。
- 雾都镇区、侦探事务所委托和“失踪调查小队”链一形成探索闭环。
- `ip_mapping` 覆盖 22 途径、7 教会和主要地名，公开文本可原创化替换。
- 数据 schema 冻结，服务端多人一致性矩阵通过。
- MVP 内测可连续运行，不依赖 M4 的灰雾、塔罗会或高序列系统。

## 已有预研

- 观众序列 9-8：情绪读取、行为预判、表层读心和心理暗示。
- 猎人序列 9-8：追踪、捕兽夹、荒野感知、挑衅、激怒和战斗意志。
- 通用仪式底座：多方块阵列、状态机、评分和五档失败结果。
- 这些实现需要在 M2 统一数据契约、多人验收和数值标准后才能计入完成度。

## M2-1 五途径低序列

- 补齐观众、猎人序列 7。
- 实现偷盗者、学徒序列 9-7。
- 统一魔药、能力、扮演、晋升和失控的数据驱动 schema。
- 建立五途径同序列 DPS / 生存 / 功能雷达对标。

## M2-2 雾都与委托

- 雾都镇区作为 MVP 社会层入口。
- 侦探事务所提供可重复委托和调查任务。
- 报纸传闻连接世界事件、材料线索和组织动向。
- “失踪调查小队”作为第一条完整调查链。

## M2-3 原创化与资产门禁

- 完成术语映射、素材来源和公开发布检查表。
- 资源 ID 使用稳定原创代号，玩家可见名称通过语言文件替换。
- 禁止官方插画、小说原文、提取资源和来源不明资产。
- 专用美术不足时允许原版回退，但必须记录替换清单。

## M2-4 多人和数据冻结

- 晋升、灵性、污染、冷却、仪式、坩埚和失控均由服务端权威处理。
- 验证重连、死亡、跨维度、区块卸载和 C2S 重放/越权。
- 冻结 M2 数据字段、注册 ID 和网络协议兼容策略。
- 输出 MVP 内测清单和已知限制。

## 明确延期

- 24 件封印物、6 个世界事件、值夜者线和经济基础属于 M3。
- 灰雾源堡、塔罗会、序列 6-5 和晋升仪式属于 M4 / EP1。
