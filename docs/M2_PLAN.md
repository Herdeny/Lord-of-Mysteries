# M2 执行计划 — 五途径序列 9-7 与 MVP 内测

> 设计基线：Project Mystery v0.8
>
> 前置门禁：M1 占卜家序列 9-7 一小时纵切通过
> 路线来源：`roadmap.json`

## 验收门槛

- 占卜家、观众、猎人、偷盗者、学徒五条途径均完成序列 9-7。
- 雾都镇区、侦探事务所委托和“失踪调查小队”链一形成探索闭环。
- `ip_mapping` 覆盖 22 途径、7 教会和主要地名，公开文本可原创化替换。
- 数据 schema 冻结，服务端多人一致性矩阵通过。
- MVP 内测可连续运行，不依赖 M4 的灰雾、塔罗会或高序列系统。

- 通用仪式底座：多方块阵列、状态机、评分和五档失败结果。
- 22 途径 `pathways_master.csv` 与自动生成的途径目录。
- `recipes_master.csv`、`gen_datapack.py` 和 Gradle 生成/校验门禁。
- 空白手稿、神秘墨水、配方残页、报纸与四条调查传闻。
- 九章神秘学手账；“钱与生计”章节由 M2 传闻解锁。
- 偷盗者序列 9：魔药、妙手空空、潜影步、应急脱身、三项扮演与窃影失控体。
- 学徒序列 9：魔药、灵性视野、小空间戏法、知识副本、三项扮演与裂门失控体。
- 偷盗者序列 8-7：换位、假身脱逃、符文解析、开锁、痕迹抹除和六项扮演。
- 学徒序列 8-7：空间转移、知识链接、镜门、星象占卜、星光庇护和六项扮演。
- 五途径 9-7 已形成代码纵切；仍需 M2 统一数据契约、多人验收和数值标准后才能计入里程碑完成度。

## M2-2 雾都与委托

- 雾都镇区作为 MVP 社会层入口。
- 侦探事务所提供可重复委托和调查任务。
- 报纸传闻连接世界事件、材料线索和组织动向。
- “失踪调查小队”作为第一条完整调查链。
- 首批报纸、手稿和配方残页只建立知识入口，不提前宣称镇区或委托完成。

## M2-0 数据生产线（已完成首批）

- 人工维护 `docs/pathways_master.csv` 与 `docs/recipes_master.csv`。
- `scripts/gen_datapack.py` 生成 22 条途径目录、11 个配方与生成清单，共 34 个资源。
- `processResources` 自动生成，`checkProjectMysteryData` 执行 `--check`。
- 后续同构 JSON 必须优先进入主表和生成器，避免手写数据漂移。

## M2-3 原创化与资产门禁

- 完成术语映射、素材来源和公开发布检查表。
- 资源 ID 使用稳定原创代号，玩家可见名称通过语言文件替换。
- 禁止官方插画、小说原文、提取资源和来源不明资产。
- 专用美术不足时允许原版回退，但必须记录替换清单。

## M2-4 多人和数据冻结

- 晋升、灵性、污染、冷却、仪式、坩埚和失控均由服务端权威处理。
- 验证重连、死亡、跨维度、区块卸载和 C2S 重放/越权。
- 冻结 M2 数据字段、注册 ID 和网络协议兼容策略。
- 输出 MVP 内测清单和已知限制。
