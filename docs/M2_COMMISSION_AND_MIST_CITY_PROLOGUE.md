# M2 雾都前哨、委托与失踪调查小队序章

> 版本：0.8.3-1.20.1  
> 设计基线：Project Mystery v0.8  
> 状态：M2 可玩预研，不计为 M2 正式验收完成

## 本批目标

本批把 v0.8 的“正常世界”从报纸和传闻推进到第一段可玩的社会循环：

1. 玩家能找到一个稳定的雾都入口。
2. 委托不硬编码在物品或命令中，而由数据包定义。
3. 第一单低风险寻猫委托承担新手导航。
4. “失踪的调查小队”形成可从接取推进到结算的序章纵切。
5. 报酬、完成记录、冷却和任务进度在死亡、重登与服务器重启后保留。

## 雾都前哨

- `/pm mistcity` 指向世界种子确定的前哨区块，距离出生区约 3–5 区块。
- 目标区块首次加载时生成石砖平台、木质雨棚、照明、档案台和委托板。
- 旧存档不需要重新开图；新的 SavedData 会在目标区块加载时补生成前哨。
- 右键委托板会根据玩家完成记录接取推荐案件；有进行中案件时用于查看或结算。
- 当前结构是 MVP 社会层入口，不代表完整雾都镇区、报社、教堂、车站和贫民巷已经完成。

## 第一单：走失的煤球

1. 在前哨右键委托板接取寻猫启事。
2. 使用 `/pm camp` 前往最近的废弃调查员营地。
3. 接近营地中的持久化黑猫“煤球”。旧存档若没有该实体，会在任务推进时补生成。
4. 返回前哨右键委托板结算。
5. 获得 24 便士账本收入、3 点事务所声望和一份雾都晨报。

这条委托把出生区、雾都入口、调查营地和原有炼药流程连接起来，不要求玩家先成为非凡者。

## 失踪的调查小队·序章

序章由七个数据驱动步骤组成：

| 步骤 | 目标 | 当前实现 |
|---|---|---|
| 1 | 接取报社委托 | 委托板 / `talk_npc:press_clerk` |
| 2 | 确认出发记录 | 进入雾都前哨 |
| 3 | 找到最后营地 | 进入废弃调查员营地范围 |
| 4 | 取得证据 | 染血调查笔记，旧营地提供任务保底 |
| 5 | 获得调查资格 | 任一首发途径达到序列 9 |
| 6 | 清除追踪者 | 击败三只特殊神秘生物或失控体 |
| 7 | 返回结算 | 前哨委托板 / `talk_npc:nighthawk_contact` |

结算获得 144 便士、10 点事务所声望和“烧焦的名单”。名单解锁链二线索知识，但链二尚未开放。

## 数据包 schema

### `commissions/`

每个委托定义以下稳定字段：

- `id`、`title_key`、`summary_key`
- `board`：允许发布该委托的委托板类型
- `level_range`：内容等级范围
- `solutions`：至少两种允许解法
- `reward.pence` 与 `reward.reputation`
- `quest_chain`：必须引用存在的任务链 ID
- `cooldown_hours` 与 `repeatable`

当前文件：

- `lost_cat.json`
- `missing_investigation_squad.json`

### `quests/`

任务链由 `steps[]` 顺序组成，每步包含：

- `id`
- `guidance_key`
- `objective.type`
- 可选 `objective.target`
- 可选 `objective.count`

v0.8 冻结 15 类目标名称：`talk_npc`、`enter_structure`、`pickup`、`encounter`、
`reach_sequence`、`rescue`、`survive_waves`、`deliver`、`brew_quality`、
`divine_success`、`ritual_outcome`、`escort`、`collect_set`、
`reputation_reach`、`custom_callback`。未知类型会被 Java 解析器和构建脚本共同拒绝。

## 命令

| 命令 | 作用 |
|---|---|
| `/pm mistcity` | 定位雾都前哨 |
| `/pm commission` | 查看当前委托 |
| `/pm commission list` | 列出数据包中的委托 |
| `/pm commission accept <id>` | 接取指定委托，例如 `lost_cat` |
| `/pm commission status` | 查看账本、步骤和计数目标 |
| `/pm commission abandon` | 放弃当前委托 |

调查委托书也可右键查看状态，玩家不必记住命令。

## 持久化与防错

- Capability schema 升级到 11。
- 保存便士余额、活动委托、活动任务链、步骤、计数进度、接取时间、完成集合和冷却表。
- 死亡复制沿用现有 Capability 克隆流程。
- 委托或任务链数据缺失时阻止接取，不写入半成品状态。
- 非重复委托由完成集合阻止重复刷取。
- `scripts/gen_datapack.py --check` 同时验证生成资源、委托、任务链、目标枚举和跨文件引用。

## 当前边界

- 这是“失踪调查小队”的可玩序章，不是设计文档中的完整 6–10 小时任务链。
- 废弃教堂、神秘学家小屋、真假配方鉴定、幸存记者 NPC、三解救援、护送和夜袭防守仍待实现。
- 账本已按便士记录并按金镑/苏勒/便士显示，但实体硬币、银行、当铺和通胀预算池尚未实现。
- `coop.shared_progress` 和 `max_party` 已进入 schema；真正的队伍共享进度将在多人一致性批次实现。
- M1 真实一小时、死亡重连和专用服务器验收仍是当前正式里程碑门禁。

## 验证清单

- 新世界和旧存档都能通过 `/pm mistcity` 找到或生成前哨。
- 委托板按顺序推荐寻猫和失踪小队序章。
- 寻猫可在普通人状态完成。
- 序章必须达到序列 9，并准确累计三次特殊生物击杀。
- 死亡、跨维度、退出重登后步骤和余额保持。
- 数据包 `/reload` 后委托与任务链仍可读取。
- 所有文案在 `zh_cn` 与 `en_us` 均存在。
