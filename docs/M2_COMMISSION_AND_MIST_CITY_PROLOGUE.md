# M2 雾都委托与失踪调查小队第一阶段

> 版本：0.9.18-1.20.1
> 设计基线：Project Mystery v0.9
> 状态：v0.9 调查与生活阶段的可玩迁移资产，不计为 M2 正式验收完成

## 本批目标

本批把 v0.8 的“正常世界”从报纸和传闻推进到第一段可玩的社会循环：

1. 玩家能找到一个稳定的雾都入口。
2. 委托不硬编码在物品或命令中，而由数据包定义。
3. 第一单低风险寻猫委托承担新手导航。
4. “失踪的调查小队”形成从报社接取、调查、救援、护送到夜袭结算的第一阶段纵切。
5. 报酬、完成记录、冷却和任务进度在死亡、重登与服务器重启后保留。
6. 旧世界能补生成前哨、教堂、邪教营地和任务 NPC。
7. 记者救援提供强攻、潜入、占卜三条服务器权威路线。
8. 神秘学家小屋与真假配方案件形成第三个可玩社会调查循环。
9. 服务端权威调查板允许玩家先比较案件、报酬、步骤和锁定原因，再主动接取或放弃。

## 雾都前哨

- `/pm mistcity` 指向世界种子确定的前哨区块，距离出生区约 3–5 区块。
- 目标区块首次加载时生成石砖平台、木质雨棚、照明、档案台和委托板。
- 旧存档不需要重新开图；新的 SavedData 会在目标区块加载时补生成前哨。
- 右键调查板打开分页案件界面；界面显示摘要、报酬、状态、当前步骤、目标进度与下一步指引，不再自动接取推荐案件。
- 站在调查板附近可用 `/pm commission board` 重新打开界面；服务端拒绝远程伪造的接取或放弃请求。
- 前哨会补生成《雾都晨报》办事员和值夜者联络员；二者均以右键交互推进服务器任务。
- 当前结构是 MVP 社会层入口，不代表完整雾都镇区、车站和贫民巷已经完成。

## 第一单：走失的煤球

1. 在前哨右键委托板接取寻猫启事。
2. 使用 `/pm camp` 前往最近的废弃调查员营地。
3. 接近营地中的持久化黑猫“煤球”。旧存档若没有该实体，会在任务推进时补生成。
4. 返回前哨右键委托板结算。
5. 获得 24 便士账本收入、3 点事务所声望和一份雾都晨报。

这条委托把出生区、雾都入口、调查营地和原有炼药流程连接起来，不要求玩家先成为非凡者。

## 失踪的调查小队·第一阶段

为兼容已经保存的七步序章进度，新内容追加在原步骤之后。任务现在由十三个数据驱动步骤组成：

| 步骤 | 目标 | 当前实现 |
|---|---|---|
| 1 | 接取报社委托 | 委托板领取后与报社办事员交谈 |
| 2 | 确认出发记录 | 进入雾都前哨 |
| 3 | 找到最后营地 | 进入废弃调查员营地范围 |
| 4 | 取得证据 | 染血调查笔记，旧营地提供任务保底 |
| 5 | 获得调查资格 | 任一首发途径达到序列 9 |
| 6 | 清除追踪者 | 击败三只特殊神秘生物或失控体 |
| 7 | 取得继续调查许可 | 与值夜者联络员交谈 |
| 8 | 搜查废弃教堂 | `/pm case` 导航与轻量程序结构 |
| 9 | 抵达邪教救援营地 | `/pm case` 导航与独立战利品 |
| 10 | 救出幸存记者 | 选择强攻/潜入/占卜路线后右键记者艾琳，获得记者证 |
| 11 | 护送返回前哨 | 记者寻路跟随，距离过远时安全追赶 |
| 12 | 三波夜袭防守 | 夜间、前哨范围、逐波清场后推进 |
| 13 | 最终结算 | 与值夜者联络员交谈 |

结算获得 144 便士、10 点事务所声望和“烧焦的名单”。名单解锁链二线索知识，但链二尚未开放。

### 调查地点与 NPC

- 废弃教堂距离出生区约 6–8 区块，包含染血笔记、配方残页、守护符和提灯等调查战利品。
- 邪教救援营地距离出生区约 10–13 区块，包含铁栏囚区、仪式残留、材料箱和幸存记者。
- 神秘学家小屋距离出生区约 8–10 区块，包含鉴定师、书架、试剂台和调查战利品。
- 三名 NPC 复用原版村民渲染，但拥有持久化标签、自定义名称、无交易任务交互和防误杀保护。
- 记者 UUID 写入玩家 Capability；护送期间由服务器驱动寻路，跨距离过大时传送到玩家附近避免区块卸载卡死。
- 夜袭生成的追兵带玩家 UUID 标签，只统计对应委托的三波敌人，不会把自然生成怪物误算为任务波次。

## 数据包 schema

### `commissions/`

每个委托定义以下稳定字段：

- `id`、`title_key`、`summary_key`
- `board`：允许发布该委托的委托板类型
- `level_range`：内容等级范围
- `solutions`：至少两种允许解法
- `reward.pence` 与 `reward.reputation`
- `quest_chain`：必须引用存在的任务链 ID
- `cooldown_hours` 与 `repeatable`

当前文件：

- `lost_cat.json`
- `missing_investigation_squad.json`
- `counterfeit_formula.json`，并通过 `prerequisites` 依赖失踪小队案件。

### `quests/`

任务链由 `steps[]` 顺序组成，每步包含：

- `id`
- `guidance_key`
- `objective.type`
- 可选 `objective.target`
- 可选 `objective.count`

v0.8 冻结 15 类目标名称：`talk_npc`、`enter_structure`、`pickup`、`encounter`、
`reach_sequence`、`rescue`、`survive_waves`、`deliver`、`brew_quality`、
`divine_success`、`ritual_outcome`、`escort`、`collect_set`、
`reputation_reach`、`custom_callback`。未知类型会被 Java 解析器和构建脚本共同拒绝。

## 命令

| 命令 | 作用 |
|---|---|
| `/pm mistcity` | 定位雾都前哨 |
| `/pm city` | 显示报社、事务所和巡警岗亭服务目录 |
| `/pm case` | 同时定位废弃教堂与邪教救援营地 |
| `/pm commission` | 查看当前委托 |
| `/pm commission board` | 在调查板附近打开可视化案件界面 |
| `/pm commission list` | 列出数据包中的委托 |
| `/pm commission accept <id>` | 接取指定委托，例如 `lost_cat` |
| `/pm commission status` | 查看账本、步骤和计数目标 |
| `/pm commission abandon` | 放弃当前委托 |
| `/pm commission approach assault|stealth|divination` | 在记者步骤锁定救援路线 |
| `/pm formula inspect` | 使用灵性或试剂检查密封卷宗 |
| `/pm formula verdict authentic|forged` | 提交真假配方结论 |
| `/pm party` | 查看队伍持久调查账本、步骤、人数和更新时间 |
| `/pm party sync` | 中途加入队伍案件或追赶离线期间的最新步骤 |

调查委托书也可右键查看状态；日常游玩优先使用调查板界面，命令保留给无障碍、诊断和服务器管理场景。

## 持久化与防错

- Capability schema 升级到 15，并在加载时钳制非法值、清理损坏引用与非法救援路线。
- 前哨 SavedData 独立保存服务区版本 1；旧世界前哨区块加载后幂等补建事务所和巡警功能亭。
- 保存便士余额、活动委托、任务步骤、计数进度、记者 UUID、夜袭活动波次、下一波时间、完成集合和冷却表。
- 主世界队伍调查账本额外保存共享步骤、路线、成员和个人结算状态，已登记成员上线后只向更远进度追赶。
- 死亡复制沿用现有 Capability 克隆流程。
- 委托或任务链数据缺失时阻止接取，不写入半成品状态。
- 非重复委托由完成集合阻止重复刷取。
- `scripts/gen_datapack.py --check` 同时验证生成资源、双语键、委托、任务链、目标枚举、合作人数和跨文件引用。

## 当前边界

- 这是“失踪调查小队”的可玩第一阶段，不是设计文档中的完整 6–10 小时精制任务链。
- 调查板列表与案件操作已完成；设计稿中的证据节点图、假说连线和多人拖拽协作属于后续 M2 批次。
- 神秘学家小屋、真假配方鉴定和三解已形成轻量可玩实现；正式潜入 AI、警戒系统与剧情演出仍待实现。
- 账本已按便士记录并按金镑/苏勒/便士显示，但实体硬币、银行、当铺和通胀预算池尚未实现。
- `coop.shared_progress` 已支持同记分板队伍 2–4 名成员同步目标与离线追赶；超过 `max_party` 自动降级为单人。
- 护送和夜袭按 UUID 选择唯一在线协调者；记者、波次和路线写入持久账本。换队/退队残留已自动清理，正式队伍 GUI、权限和队名复用管理仍未实现。
- M1 两小时合同、双启动专服与连续性证据已完成；真人两小时生存和平衡记录继续作为发布质量回归。

## 验证清单

- 新世界和旧存档都能通过 `/pm mistcity` 找到或生成前哨。
- 调查板能分页显示三个案件及可接取、进行中、已完成、锁定和冷却状态。
- 调查板附近可接取/放弃，远程或伪造案件 ID 的请求由服务端拒绝。
- 寻猫可在普通人状态完成。
- 调查链必须达到序列 9，并准确累计三次特殊生物击杀。
- `/pm case` 能定位两个种子确定的调查地点，旧世界加载区块后自动补生成。
- 报社办事员、值夜者联络员、记者、鉴定师、事务所文员和巡警右键交互不会打开村民交易界面。
- 事务所普通交互打开调查板，潜行交互准确扣除 18 便士并发放调查补给包。
- 巡警普通交互显示风险/案件状态，潜行交互准确扣除 28 便士并钳制恢复后的压力和污染。
- 记者能跟随玩家抵达前哨，三波追兵全部清除后才允许最终结算。
- 强攻守卫按队伍隔离；潜入/占卜门槛、消耗和路线持久化准确生效。
- 完成失踪小队后可进入小屋、取得卷宗、鉴定、提交结论并结算第三份委托。
- 2–4 名同记分板队员共享目标且只由一名协调者驱动护送/夜袭；不同队与超过上限的队伍不得串档。
- 已登记成员离线后由队友推进，上线自动追赶；中途加入使用 `/pm party sync`，每名成员独立结算且不得重复领奖。
- 死亡、跨维度、退出重登后步骤和余额保持。
- 数据包 `/reload` 后委托与任务链仍可读取。
- 所有文案在 `zh_cn` 与 `en_us` 均存在。
- 完整专服与多人矩阵见 [`DEDICATED_SERVER_TEST_MATRIX.md`](DEDICATED_SERVER_TEST_MATRIX.md)。
- 城市服务完整规则见 [`M2_CITY_SERVICE_DESKS.md`](M2_CITY_SERVICE_DESKS.md)。
