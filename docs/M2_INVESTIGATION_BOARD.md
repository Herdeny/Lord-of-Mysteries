# M2 调查板玩家手册

> 适用版本：`0.9.9-1.20.1` · Minecraft 1.20.1 · 网络协议 12 · Capability schema 19
>
> 定位：M2 调查与生活的正式案件入口；当前包含案件公告、只读证据档案、服务端推理页与玩家假说工作区，不代表可自由拖拽的节点编辑器已经完成。

## 最短上手路线

1. 完成 M1 占卜家序列 9–7 主纵切。
2. 输入 `/pm mistcity` 前往雾都前哨。
3. 右键调查板，比较案件摘要、报酬、锁定条件和冷却。
4. 选择可接取案件并确认；界面会立即显示当前步骤、目标进度和下一步指引。
5. 点击顶部按钮依次切换“证据档案”和“关联推理”，查看证据状态、已发现关系、当前理论与唯一下一步。
6. 关系行显示稳定 `#关系ID`；输入 `/pm case hypothesis` 查看可用关系，再在调查板附近提出并检验自己的支持、矛盾或指向假说。
7. 误判会增加压力与推理负担；用正确立场重新检验，或等待后点击“复议”逐点恢复。
8. 关键委托纸或真假配方卷宗意外丢失时，在调查板附近输入 `/pm case recover` 安全补回。
9. 返回调查板可刷新状态或放弃案件；也可在调查板附近输入 `/pm commission board`。
10. 结案前处理未解决推理负担；结案后用 `/pm case debrief <案件>` 重看持久化复盘。

## 界面信息顺序

调查板按“当前行动优先”组织信息：

1. 顶部显示便士余额与当前进行中的案件。
2. 当前案件显示步骤、计数进度和下一步指引。
3. 案件列表按数据包定义排序并分页，避免同屏信息过载。
4. 每项显示标题、摘要、便士/声望报酬与状态按钮。
5. 只有服务端确认后，界面才会刷新为新的权威状态。

证据页保持相同的案件上下文，并按“已记录数量 → 结论准备度 → 证据标题 → 说明 → 状态”展示；超过四条时分页。关联推理页按“完整度 → 证据统计 → 下一步 → 当前理论 → 玩家假说 → 已发现关系”展示，并区分收集中、关联中、可结论三个阶段。玩家假说区显示关系 ID、立场、状态、说明与 0–3 点推理负担。

## 五种案件状态

| 状态 | 含义 | 玩家操作 |
|---|---|---|
| 可接取 | 前置、完成记录和冷却均允许 | 可点击接取 |
| 进行中 | 当前正在追踪该案件 | 查看步骤或放弃 |
| 已完成 | 不可重复案件已结算 | 只读查看 |
| 已锁定 | 前置未完成、已有其他案件或定义不允许 | 按提示先完成条件 |
| 冷却中 | 可重复案件尚未结束冷却 | 等待后再次接取 |

## 命令与无障碍入口

| 命令 | 用途 |
|---|---|
| `/pm commission board` | 在调查板附近打开界面 |
| `/pm commission status` | 以聊天文本查看当前步骤 |
| `/pm commission list` | 以聊天文本列出案件 |
| `/pm commission accept <id>` | 服务器管理或无界面场景接取 |
| `/pm commission abandon` | 服务器管理或无界面场景放弃 |
| `/pm case sites` | 查看三个固定案件现场和距离 |
| `/pm case analyze` | 以聊天文本查看阶段、理论、下一步和已发现关系 |
| `/pm case hypothesis` | 查看当前假说、推理负担和已揭示关系 ID |
| `/pm case hypothesis propose <supports\|contradicts\|leads_to> <关系ID> <说明>` | 在调查板附近提出自定义假说 |
| `/pm case hypothesis test\|reconsider\|clear` | 检验、复议恢复，或仅清除当前草稿 |
| `/pm case archive` | 查看已结案案件、评级、总体完成度与平均分 |
| `/pm case debrief [案件]` | 查看档案摘要或指定案件的四维评分、耗时、路线和建议 |
| `/pm case recover` | 在真实调查板附近幂等补回缺失的关键案件物品 |
| `/pm party` | 查看持久队伍调查账本 |
| `/pm party sync` | 中途加入或追赶离线进度 |

## 服务端安全模型

- 客户端只发送“接取哪个案件”或“放弃当前案件”的意图，不发送最终余额、步骤或完成状态。
- 服务端重新检查玩家必须在真实调查板水平 6 格、垂直 4 格范围内。
- 接取前重新检查案件 ID、前置、活动案件、完成集合、可重复性和冷却。
- 放弃前重新检查活动案件；请求同时受现有 C2S 频率门禁保护。
- S2C 快照最多携带 32 个案件条目和每案 32 条关系，字段使用有界字符串、钳制计数和固定枚举。
- 证据、关系、完整度、理论和下一步只从任务进度、知识记录和服务端卷宗 NBT 推导；客户端没有“提交证据”或“提交结论”消息。
- 未揭示关系不会发送给客户端；提出、检验、复议和清除假说时，服务端重新校验真实调查板、活动案件、稳定关系 ID、关系可见性、冷却和 160 字有界说明。
- 错误检验增加 4 点压力与最多 3 点推理负担；正确检验或 30 秒后复议可恢复 1 点，单纯清草稿不能免除代价。
- `/pm case recover` 重新检查活动案件、真实调查板距离、背包缺失和允许步骤；不会推进案件、发放奖励或覆盖现有物品。
- 结案评分只在服务端确认可结算后生成，客户端不能提交分数；复盘不修改原有奖励。
- 网络协议为 12，14 个现有消息 ID 保持不变，调查板快照扩展稳定关系 ID 与玩家假说字段。

- 6 项 Forge GameTest，包括真实方块邻近成功与远程拒绝。
- `scripts/check_m2_investigation.py` 固定调查板入口、协议 12、14 个消息、证据档案、关联推理、玩家假说、误判恢复、案件物品恢复、每日新闻、状态翻译与服务端邻近门禁。
