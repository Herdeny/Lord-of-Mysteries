## 新增运行时基础

### 公共内容元数据

`ContentMetadata` 统一解析：

- `schemaVersion`
- `canonStatus`
- `sourceTier` 与 `sourceRefs`
- `spoilerLevel`
- `knowledgeGate`
- `links`
- `implementationState`
- `canonStatus` 与 `sourceTier` 的来源策略约束

运行时委托和任务链缺少必要元数据时会拒绝加载；构建图谱还会验证途径和序列数据。

### 特性 bundle

每次序列晋升都会向玩家账本写入一层特性，包含途径、序列、纯度、印记、污染和来源哈希。旧 schema 15 存档在首次加载时根据现有途径/序列生成 legacy bundle。永久失控当前保留账本而清除玩家途径身份，为后续特性提取和掉落实体预留可信来源。

### 扮演 v2 基础

玩家数据新增原则理解、角色过度认同、反思次数和最近反思日。高新颖度扮演提高原则理解，机械重复与高压晋升提高过度认同；`/pm reflect` 每游戏日一次，降低压力与过度认同并增加原则理解。状态包和非凡者档案同步显示新字段。

### 首次部署存档安全

世界加载前会为 schema 16 创建一次相关文件快照；Capability 使用命名 DataFix 链迁移无版本旧档和 schema 15，并把原始 NBT、迁移历史、非法 ID、损坏 bundle 与未来 schema 保存到 `migration_backups` / `orphaned_entries`。既有 M1 试炼和 M2 调查状态继续使用同一 Capability，不需要清档或重新开档。

### 状态分区与生命周期同步

玩家数据按 Core、Knowledge、Social、Endgame 四区跟踪运行时 dirty mask。登录、重生和维度切换立即下发独立核心摘要；每秒检查变化并每 5 秒校正。摘要不会打开状态 GUI，也不会发送完整知识集合。202 项 JUnit 与 3 项真实 Forge GameTest 验证分区、协议、Capability Clone、schema 迁移和未来数据隔离。
