## 新增运行时基础

### 公共内容元数据

`ContentMetadata` 统一解析：

- `schemaVersion`
- `canonStatus`
- `sourceTier` 与 `sourceRefs`
- `spoilerLevel`
- `knowledgeGate`
- `links`
- `implementationState`
- `canonStatus` 与 `sourceTier` 的来源策略约束

运行时委托和任务链缺少必要元数据时会拒绝加载；构建图谱还会验证途径和序列数据。

### 特性 bundle

每次序列晋升都会向玩家账本写入一层特性，包含途径、序列、纯度、印记、污染和来源哈希。旧 schema 15 存档在首次加载时根据现有途径/序列生成 legacy bundle。永久失控会把精确 bundle 转移到失控实体，并在实体死亡时生成带完整载荷的破碎非凡特性；可恢复崩溃只保留通用象征碎片，避免复制或凭空丢失。

### 扮演 v2 基础

玩家数据包含原则理解、角色过度认同、反思次数和最近反思日。首次服用魔药会发放扮演身份卡与反思日志；身份卡绑定当前途径角色，`/pm reflect` 或日志交互每游戏日一次降低压力与过度认同并增加原则理解。身份、反思与城市生活均进入两小时 M1 验收记录。

### 首次部署存档安全

世界加载前会为 schema 17 创建一次相关文件快照；Capability 使用命名 DataFix 链迁移无版本旧档、schema 15 和 schema 16，并把原始 NBT、迁移历史、非法 ID、损坏 bundle 与未来 schema 保存到 `migration_backups` / `orphaned_entries`。恢复工具支持预演、清单校验、恢复前安全快照、精确替换和路径穿越拒绝，既有 M1/M2 存档无需清档。

### 状态分区与生命周期同步

玩家数据按 Core、Knowledge、Social、Endgame 四区跟踪运行时 dirty mask。登录、重生和维度切换立即下发独立核心摘要；每秒检查变化并每 5 秒校正。摘要不会打开状态 GUI，也不会发送完整知识集合。207 项 JUnit 与 6 项真实 Forge GameTest 验证分区、协议、Capability Clone、schema 迁移、未来数据隔离、精确特性载荷、两小时试炼状态往返和调查板邻近门禁。

### M2 调查板

右键雾都调查板打开服务端权威案件快照；列表显示摘要、报酬、状态、当前步骤和计数进度。客户端只发送接取/放弃意图，服务端重新校验真实方块距离、案件 ID、前置、进度、完成记录与冷却。协议 9 以追加 ID 注册 14 个消息，既有 ID 不重排。
