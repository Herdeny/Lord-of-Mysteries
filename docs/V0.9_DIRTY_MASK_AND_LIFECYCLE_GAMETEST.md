# v0.9 状态分区与生命周期 GameTest

> 适用版本：0.9.2-1.20.1 · Capability schema 16 · 内容 schema v4 · 网络协议 8

## 目标

本批完成 v0.9 M0 的玩家状态分区和首批 Minecraft 运行时回归。重点不是增加表面内容，而是让旧 M1/M2 迁移资产在登录、重生、维度切换、保存和版本迁移后仍能可靠工作，并避免每次变化都发送完整玩家数据。

## 四区 dirty mask

| 分区 | 当前覆盖 | 同步策略 |
|---|---|---|
| Core | 途径、序列、灵性、消化、污染、压力、特性 bundle、能力状态/冷却、扮演身份摘要、schema/迁移元数据 | 登录/重生/换维度立即同步；每秒检查；5 秒校正 |
| Knowledge | `knownKnowledge`、扮演历史、扮演计数 | 置脏并保留，等待知识手账专用增量通道消费 |
| Social | 钱、委托、任务链、完成记录、冷却、组织声望 | 置脏并保留，等待调查/经济专用增量通道消费 |
| Endgame | M1 试炼、连续性证据、阶段时间与统计 | 置脏并保留，等待验收/终局专用增量通道消费 |

现有 `PlayerMysteryData` 仍保留公共字段以兼容已经实现的几十处能力、任务和引导写入。运行时指纹在下一次检查时识别字段和集合原地变化，因此无需一次性重写全部调用点；未来字段收敛为 setter 或分区对象后，可直接在写入处 `markDirty`。

dirty mask 只存在于运行时，不进入玩家 NBT。新建、加载与 Clone 后四区都标记为 dirty，保证新连接先收到权威状态；某一通道确认后只清除自己的 bit，不会误清除其他待同步分区。

## 核心摘要同步

- 新增独立 `PlayerMysterySummaryS2CPacket`，不复用会打开 GUI 的完整状态包。
- 摘要只包含 HUD/即时反馈需要的 Core 字段，不发送知识全集。
- 精确消化值继续服从 `show_exact_digestion` 服务端配置；关闭时发送隐藏标记。
- 客户端只写入 `ClientMysteryState` 只读缓存，断开连接时清空。
- 登录、重生和维度切换强制同步；服务端每 20 tick 检查一次 Core dirty；即使无变化也每 100 tick 校正一次。
- 无真实 Netty channel 的测试玩家不会发包，防止 Forge 测试连接误崩。

SimpleChannel 协议由 7 升至 8，消息 ID 保持 0–11 显式、唯一、连续。协议严格匹配，因此 0.9.2 客户端与旧协议服务端不会静默混连。

## 自动化回归矩阵

### JUnit 5：202 项

本批新增 6 项：四区原地修改检测、分区确认隔离、加载/Clone 初始全脏、dirty/5 秒校正策略、摘要字段与精确消化隐藏、协议 8 严格匹配。

### Forge GameTest：3 项

| 测试 | 运行时证据 |
|---|---|
| Capability Clone | 真实 `ServerPlayer` 实例附着 capability；途径、能力、知识、经济和试炼状态复制到新实体 |
| schema 15 Provider 往返 | 旧 NBT 经正式 DataFix 迁到 schema 16，生成特性 bundle、迁移历史和恢复备份后重新保存 |
| future schema 隔离 | 未来字段不被旧代码解释或静默丢弃，完整载荷进入 `orphaned_entries` |

`runGameTestServer` 必须显示 `3 tests are now running` 和 `All 3 required tests passed`。仅退出码为 0 但报告 `0 tests` 不算通过。

## CI 顺序

```text
设计源/生成器/内容图/合同/资源门禁
→ clean build + 202 JUnit
→ runGameTestServer + 3 required GameTests
→ Forge 专服诊断、命令、save-all flush、干净停服
```

本地完整验证：

```bash
python scripts/sync_project_metadata.py --check
./gradlew clean build
./gradlew runGameTestServer
python scripts/run_server_smoke.py --timeout 180
```
