## 3. 数据生产决策 D9

人工维护：

- `docs/pathways_master.csv`
- `docs/recipes_master.csv`

自动生成：

- `src/generated/resources/data/lord_of_mysteries/pathway_catalog/*.json`
- `src/generated/resources/data/lord_of_mysteries/recipes/*.json`
- `src/generated/resources/data/lord_of_mysteries/project_mystery/generated_manifest.json`

命令：

```bash
python scripts/gen_datapack.py
python scripts/gen_datapack.py --check
./gradlew clean build
```

Gradle 的 `processResources` 和 `check` 已接入生成器。后续物品、能力、任务和事件数据必须优先
扩展 CSV/schema，而不是继续批量手写同构 JSON。

## 4. M1 指引与验收

- 原调查员手稿升级显示为“神秘学手账”，保留稳定物品 ID。
- `/pm handbook` 显示九章目录，`/pm handbook <1-9>` 阅读章节。
- 安全、成为非凡者、风险和途径章节默认开放。
- 扮演/仪式随成为非凡者开放，占卜随占卜家开放，生计随 M2 传闻开放。
- 灰雾章节在 M4 邀请知识出现前保持锁定。
- `/pm trial` 包含 6 项核心目标与 4 项连续性门禁；`stop` 只统计在线时间，`start` / `resume` 恢复保留记录。
- `/pm doctor` 在开测前修复玩家数据并检查定义、协议和确定性世界目标。
