# M2 城市服务台与功能区

> 适用版本：`0.9.18-1.20.1` · Minecraft 1.20.1 · 网络协议 12 · Capability schema 23 · 前哨服务区版本 2

## 玩家入口

1. 完成 M1 纵切后输入 `/pm mistcity` 前往雾都前哨。
2. 输入 `/pm city` 查看报社、侦探事务所和巡警岗亭目录。
3. 普通交互使用基础服务；按住潜行键交互使用工作、交易或恢复服务。

## 三类服务台

| 服务台 | 普通交互 | 潜行交互 | 价格或产出 |
|---|---|---|---|
| 报社职员 | 推进报社线索 | 消耗 3 张纸完成每日报社工作 | 24 便士、2 份面包、当日报纸、压力 +0、暴露 -4 |
| 侦探事务所文员 | 打开服务端权威调查板 | 完成每日事务所工作 | 22 便士、压力 +1、暴露 -2 |
| 巡警 | 查看余额、风险、暴露与活动案件 | 完成每日巡逻工作 | 28 便士、压力 +2、暴露 -3 |

潜行交互优先执行三职业工作；原有调查补给和安全室服务继续由服务逻辑保留为案件恢复入口。三种工作共享每日一次限制，不能分别刷三次。`/pm life work press|agency|patrol` 提供等价命令入口。

工资、纸张成本、压力和神秘暴露由 `CityEconomyPolicy` 统一计算，并受当前世界事件修正。报纸同时显示世界事件和暴露档位，避免玩家只能从隐藏数值猜测风险。

## 世界升级

- `MistCityOutpostSavedData` 独立保存 `service_version`，当前值为 2。
- 新世界在三个角区块可用后生成报社、侦探事务所和警局三座 7×7 正式功能建筑及连接道路。
- 旧世界先兼容补建 v1 功能亭，再幂等迁移到 v2 正式街区；已标记的 NPC 会移动到新服务点，不重复生成。
- 服务区升级不重排注册 ID 或网络消息 ID；玩家 Capability 使用 schema 23 保存三职业班次和神秘暴露。
- 当前建筑仍以原版方块提供稳定碰撞、寻路和旧世界兼容；原创模型和剧情演出属于后续精制。

## 服务端边界

- 余额检查、扣款、物品发放和风险恢复全部在服务端执行。
- 事务所仍调用调查板六格邻近门禁，不能成为远程打开案件界面的后门。
- NPC 复用原版村民渲染，但交互会被任务系统接管，不打开原版交易界面。
- 工作数值集中在 `CityEconomyPolicy`，暴露规则集中在 `MysticalExposurePolicy`，运行交互集中在 `CityLifeService` / `CityServiceDeskService`，避免命令与 NPC 路径产生两套结算。
- 世界事件、工资、纸张、压力、暴露和每日次数都由服务端读取；客户端不能自报职业、收益或事件。

## 自动验证

- 340 项 JUnit 覆盖三职业全事件矩阵、每日限制、工资/成本/压力/暴露边界、动态案件隔离、四人八小时快照、世界 SavedData 和 schema 23 迁移。
- M2 合同校验三座正式街区、12 项周指令、五档组织立场、三职业经济、六种世界事件、`service_version=2` 和旧前哨幂等迁移。
- 资源门禁校验 303 个 JSON、1343 对双语键、514 个静态引用、90 个模型和 85 物品、5 方块、8 实体。
- 完整构建运行 9 项 Forge GameTest、隔离回滚与同一世界双启动专服矩阵；其中五途径晋升与能力场景验证灵性、特性层、Provider 重启、效果、冷却和物品归属。

## 当前边界

- 这是三座正式功能街区，不是完整维多利亚城市、银行、医院或可扩张城市模拟。
- 商品和工资仍为固定服务器规则，尚未接入供需、租金、声望折扣或组织预算。
- 安全室只提供数值恢复，没有床位占用、医疗材料、排队或多人容量。
- M2 自动化范围已经闭环；四人八小时真人验收仍需记录经济净流入、并行案件、跨日工作、事件切换和重连无锁死证据。
