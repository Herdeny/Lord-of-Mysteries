# 设计基线

## 当前来源

- 设计版本：Project Mystery v0.6
- 审计日期：2026-07-04
- 用户提供源文件：`Project_Mystery_Design_Doc_v0_6.md.pdf`
- 工程基线：Minecraft 1.20.1、Forge 47.4.20、Java 17

仓库现有实现始终优先于设计文档。设计文档中的 NeoForge、新版 Minecraft API、
未注册资源 ID 和示例包名不能直接复制，必须转换为当前仓库架构。

## v0.6 主要增量

- 仪式拆分为场地阵列、材料、环境、主持者、呼名和结果结算。
- 尊名呼名、错误呼名灾变与玩家尊名注册。
- 罗塞尔日记、知识手册、金镑经济和会客法。
- 灰雾、塔罗会、多人匿名身份与祈求回应。
- 五条 MVP 途径的序列 9–4 规划，以及更多封印物、组织和世界系统。

## 路线映射

v0.6 的正式路线是 M0–M7，不再沿用旧的“M2 灰雾基础版 / M4 MVP 1.0”路线：

- M0：工程骨架。
- M1：占卜家序列 9-7 一小时纵切。
- M2：五途径序列 9-7、雾都、委托、调查链一和 MVP 内测。
- M3：24 件封印物、6 个世界事件、值夜者线和经济基础，进入 MVP 公测。
- M4：序列 6-5、晋升仪式、灰雾、塔罗会和特性完整版，对应 EP1。
- M5：序列 4、GeckoLib 神话形态、第二批四途径和组织战争，对应 EP2。
- M6：灵界维度、1 级封印物和大型城市，对应 EP3。
- M7：22 途径社区共创与 Addon API 1.0。

路线数据的唯一来源是 `roadmap.json`，完整输出见 `ROADMAP.md`。当前仓库按新门禁处于
M1；观众/猎人序列 9-8 和通用仪式底座保留为后续阶段预研，不会被删除或伪装成
M1 已验收。
