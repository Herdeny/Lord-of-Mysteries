# 设计基线

## 当前来源

- 设计版本：Project Mystery v0.6
- 审计日期：2026-07-04
- 用户提供源文件：`Project_Mystery_Design_Doc_v0_6.md.pdf`
- 工程基线：Minecraft 1.20.1、Forge 47.4.20、Java 17

仓库现有实现始终优先于设计文档。设计文档中的 NeoForge、新版 Minecraft API、
未注册资源 ID 和示例包名不能直接复制，必须转换为当前仓库架构。

## v0.6 主要增量

- 仪式拆分为场地阵列、材料、环境、主持者、呼名和结果结算。
- 尊名呼名、错误呼名灾变与玩家尊名注册。
- 罗塞尔日记、知识手册、金镑经济和会客法。
- 灰雾、塔罗会、多人匿名身份与祈求回应。
- 五条 MVP 途径的序列 9–4 规划，以及更多封印物、组织和世界系统。

## 当前实施边界

本版本完成 M2-3 的通用仪式底座：物理阵列、周期复检、材料/环境/主持者评分、
五档结算和失败风险。暂不实现依赖灰雾、组织、网络交互或高序列内容的尊名呼名、
玩家祈求、晋升饮药窗口与多人参与加成。

后续实施顺序以 `docs/M2_PLAN.md` 为准；任何功能完成后必须同步 README、Pages、
Wiki、Changelog 和 `project-status.json`。
