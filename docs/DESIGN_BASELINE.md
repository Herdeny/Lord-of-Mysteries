# 设计基线

## 当前来源

- 设计版本：Project Mystery v0.8
- 最近同步：2026-07-16
- 用户提供源文件：`Project_Mystery_Design_Doc_v0_8.md`
- 仓库执行版：`docs/Project_Mystery_Design_Doc_v0.8.md`
- 工程基线：Minecraft 1.20.1、Forge 47.4.20、Java 17

仓库现有实现始终优先于设计文档。v0.8 中的示例包名 `pm`、Forge 47.3.x、
未接入的 GeckoLib 类型、未注册资源 ID 和面向其他仓库的服务器桥接不能直接复制，
必须转换为 `lord_of_mysteries` 命名空间和当前 Forge 47.4.20 架构。

## v0.8 主要增量

- 180+ 物品、60+ 工作台配方、75 魔药配方与 13 类数据文件规范。
- D9 数据生产决策：CSV 是人工主表，`scripts/gen_datapack.py` 生成和校验 JSON。
- 22 途径目录、15 条长期可玩路线、150+ 能力和统一执行器账本。
- 九章神秘学手账、12 步教学链、情境提示与每途径首小时路书。
- 18 类新生物、34 类粒子、49 个 GeckoLib 模型目标。
- 雾都生活、报纸、委托、任务链、服务器运营与长线纪元叙事。

这些数字是长期生产目标，不代表当前版本完成度。

## 本批落实

- `docs/pathways_master.csv`：22 途径单一主表。
- `docs/recipes_master.csv`：首批 M2 工作台配方主表。
- `scripts/gen_datapack.py`：生成 22 条途径目录、11 个配方和 1 个清单，共 34 个资源。
- Gradle 构建自动执行生成与 `--check`。
- 神秘学手账九章目录和条件解锁。
- M1 验收增加“两次有效扮演”目标，当前 Capability schema 为 10。
- M2 知识经济首批物品与配方残页调查闭环。
- 五条 MVP 途径序列 9-7 已形成代码可玩预研，仍等待 M2 多人和内容门禁。

## 路线映射

- M1：占卜家序列 9-7 一小时纵切，仍是当前 `active` 里程碑。
- M2：五途径序列 9-7、雾都、委托、调查链一和 MVP 内测；允许预研，不得标记完成。
- M3：24 件封印物、6 个世界事件、组织线和经济基础。
- M4：序列 6-5、晋升仪式、特性、灰雾和塔罗会。
- M5：序列 4、GeckoLib 神话形态、第二批四途径和组织战争。
- M6：灵界、大城市、1 级封印物与更大规模内容生产。
- M7：22 途径社区共创、稳定 API 和长期生态。

路线数据唯一来源是 `roadmap.json`。真实一小时、死亡重连和专用服务器验收完成前，
不得把 `current_milestone` 改为 M2。
