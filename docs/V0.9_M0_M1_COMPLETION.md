## 结论

M0 内容基建与 M1 核心垂直切片的**代码、数据、资源和自动化实施门禁已经完成**，路线图转入 M2 调查与生活。这里的“完成”不等于 1.0 发布验证：真人两小时节奏、多人平衡、无障碍与公开长测仍需持续记录。

## M0 完成证据

- 内容图：114 个真实节点、160 条关系；同时生成孤儿、剧透迁移、兼容、本地化和资产报告。
- 数据迁移：schema 0/15 经命名 DataFix 迁至 schema 17；保留原始 NBT、迁移历史、损坏条目和未来 schema。
- 世界快照：加载世界前创建原子快照与幂等 `schema-17.complete` marker。
- 安全回滚：`scripts/restore_migration_backup.py` 只接受备份根目录的直属快照，拒绝路径穿越和符号链接；实际恢复要求 `--confirm`，并先创建恢复前安全副本。
- 回滚演练：`scripts/check_save_rollback.py` 验证 dry-run、精确覆盖、快照后新增文件移除、安全副本和路径穿越拒绝。
- 重启演练：`scripts/run_server_restart_matrix.py` 连续两次启动同一世界，核对世界种子、schema 快照、定义加载、运行诊断、保存和干净停服。
- 运行时生命周期：6 项 Forge GameTest 覆盖 Capability Clone、schema 15→17、未来数据隔离、精确特性载荷、M1 两小时状态与调查板邻近门禁。

## M1 完整玩法

1. 首次登录取得调查手册与罗盘，前往世界种子确定的调查营地。
2. 利用保证供应的坩埚和材料炼制占卜家序列 9 魔药。
3. 通过能力与扮演消化，继续推进小丑序列 8 和魔术师序列 7。
4. 首次服药自动取得扮演身份卡和反思日志；旧角色可用 `/pm recover` 补回。
5. 身份卡右键锚定途径、序列和行为边界；潜行右键在四个隐私安全预设之间切换。
6. 反思日志或 `/pm reflect` 每游戏日记录一次原则理解，降低角色过度认同与压力。
7. `/pm mistcity` 定位轻量雾都街区；生活补给提供纸张和食物，潜行交互报社职员完成每日轮班并取得便士与餐食。
8. `/pm next` 和 `/pm m1check` 会继续引导身份、反思和谋生，不会在抵达序列 7 后过早宣布完成。
9. 永久失控时，当前途径的精确 `CharacteristicBundle` 转移到失控体；击杀后掉落携带途径、序列层、印记、污染和来源哈希的破碎特性。

## 两小时九目标

`/pm trial start|resume|status|report|verify|stop|reset` 记录：

| 目标 | 要求 |
|---|---|
| 在线时长 | 144,000 tick / 两小时 |
| 调查营地 | 进入营地 24 格范围 |
| 序列推进 | 占卜家途径到达序列 7 |
| 狩猎 | 击杀至少 3 个神秘生物 |
| 有效扮演 | 至少 2 次正收益扮演 |
| 风险观察 | 压力或污染峰值至少 25 |
| 身份锚定 | 使用扮演身份卡 |
| 反思 | 完成至少 1 次反思 |
| 城市生活 | 完成至少 1 次报社轮班 |

七个阶段目标为营地 10 分钟、序列 9/8/7 分别 30/60/90 分钟、身份 100 分钟、反思 110 分钟、谋生 120 分钟。连续性证据仍独立记录重连、服务器重启、维度往返和死亡恢复。

## 恢复命令

先预览：

```bash
python scripts/restore_migration_backup.py run/world schema-17-YYYYMMDDTHHMMSSZ --dry-run
```

确认无误后恢复：

```bash
python scripts/restore_migration_backup.py run/world schema-17-YYYYMMDDTHHMMSSZ --confirm
```

恢复会覆盖快照内受保护文件，并移除快照后新增的受保护 `.dat`；所有当前文件会先复制到 `project_mystery_backups/pre-restore-*`。
