> 本项目是未获原作权利人授权的非商业同人项目。公开仓库不代表已取得《诡秘之主》相关名称、设定、角色或素材的再许可权。
>
> 本表服务于 v0.6 M2 的原创化验收：代码和数据使用稳定的中性内部 ID，玩家可见名称集中在语言文件中，以便在公开发布前统一替换。

## v0.6 路线要求

- **M1**：禁止新增不可替换的原作专有名词、官方素材或小说原文。
- **M2**：完成五途径序列 9–7、组织、地点和核心物品的映射清单。
- **M3**：公开测试前完成素材来源审计，移除未授权字体、音乐、图片和提取资源。
- **发布门槛**：所有玩家可见文本必须可通过 `lang/*.json` 独立替换，存档 ID、网络协议和 Java 包名不得绑定原作中文专名。

## 切换策略

- 代码层：`mod_id`、JSON 的 `id` 和 `ResourceLocation` 使用稳定英文内部 ID。
- 展示层：所有玩家可见文本进入语言文件，公开名称变更不得要求迁移存档或协议。
- 素材层：只接受原创、明确兼容许可或可证明获得授权的素材。
- 审核层：新增内容提案必须说明公开名称、玩法差异和素材来源。

## 提交门槛

- 禁止提交小说原文、官方插画、动画或游戏提取素材、官方 Logo、未授权音乐和字体。
- 原作术语不得耦合到存档 ID、网络协议、数据包结构或 Java 包名。
- 权利人提出可信请求时，维护者可立即隐藏、替换或移除相关内容。
- 授权边界以仓库根目录的 `NOTICE.md`、`ASSET_LICENSE.md` 和 `LICENSE` 为准。
