# 仓库健康审计与资源完整性门禁

> 适用版本：0.9.6-1.20.1
>
> 技术基线：Minecraft 1.20.1 · Forge 47.4.20 · Java 17 · Gradle 8.14.5

本页记录 0.8.9 健康审计，并追加 0.9.1 存档迁移、0.9.2 dirty mask/生命周期同步、0.9.3 M0/M1 完成门禁、0.9.4 调查板协议门禁与 0.9.5 证据/报纸契约。目标不是增加表面内容，
而是确保现有迁移资产在损坏数据、换队、重复同步、资源漂移和未来构建工具升级时仍能安全失败。

## 本轮修复

- 队伍成员离开或切换记分板队伍时，从旧调查账本移除 UUID；个人任务进度保留，不跨队串档。
- 放弃与结算按玩家 UUID、委托 ID 和任务链 ID 查找账本，不再依赖当前队伍人数或名称。
- 已完成不可重复委托不能通过 `/pm party sync` 再领奖；可重复委托同步必须通过冷却检查。
- 只有已登记且未结算成员可以推动权威账本，四人上限不能被离线旧成员或损坏 NBT 绕过。
- 无效队伍键、完成账本、损坏记者 UUID、越界步骤和目标计数会被丢弃或拒绝。
- 任务进度使用长整型安全累加，避免极端或损坏输入导致整数回绕。
- 委托与任务链的运行时解析和生成器统一校验非空、唯一性、等级、失败策略、人数、计数与溢出。
- Forge 入口改用构造上下文注入；Gradle 发布仓库移除旧 Maven DSL。`--warning-mode all` 已确认剩余提示来自 ForgeGradle 6 的 `downloadMcpConfig` / `reobfJar` 内部实现，而非项目脚本。

## 统一资源门禁

运行：

```bash
python scripts/check_resource_integrity.py
```

当前门禁验证：

| 范围 | 当前基线 |
|---|---:|
| 可解析 JSON | 266 |
| 中英成对翻译键 | 893 |
| Java/JSON 静态引用翻译键 | 374 |
| 模型 JSON | 75 |
| 注册物品 | 70 |
| 注册方块 | 5 |
| 注册实体 | 8 |

检查器会拒绝：

- 任意主资源或生成资源 JSON 语法错误；
- `en_us` / `zh_cn` 键集合漂移或静态引用缺失；
- 模型引用不存在的本 Mod 模型或纹理；
- 注册物品缺少模型/名称，注册方块缺少名称、blockstate 或模型，注册实体缺少名称。

`checkResourceIntegrity` 已加入 Gradle `check`，Build 与 Documentation Consistency 工作流也会独立执行。
`check_github_text_encoding.py` 会在 Issue/评论创建或编辑时通过 GitHub API 扫描全部协作文本，拒绝连续问号和 Unicode 替换字符，
因此代码、资源、文档与协作文本改动都不能绕过对应门禁。
