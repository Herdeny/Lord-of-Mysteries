# M1 两小时试炼追踪器

> 适用版本：`0.9.12-1.20.1` · v0.9 M1 实施完成 · Capability schema 19

追踪器只统计当前存档中的在线游戏时间，不需要管理员权限。它提供可重复的机器证据，但不替代真人节奏、无障碍和多人平衡反馈。

## 命令

| 命令 | 作用 |
|---|---|
| `/pm trial start` | 新建或恢复保留的试炼 |
| `/pm trial resume` | 与 `start` 相同，强调继续旧记录 |
| `/pm trial status` | 显示九项核心目标和四项连续性证据 |
| `/pm trial report` | 显示七个阶段里程碑和首次关键事件 |
| `/pm trial verify` | 同时核对核心目标与连续性证据 |
| `/pm trial stop` | 暂停在线计时并保留全部记录 |
| `/pm trial reset` | 清空本轮试炼记录，不清除角色长期身份/反思/谋生数据 |

## 九项核心目标

1. 在线累计 144,000 tick，即两小时。
2. 进入调查营地 24 格范围。
3. 占卜家途径到达序列 7。
4. 击杀至少 3 个神秘生物。
5. 完成至少 2 次正收益扮演事件。
6. 压力或污染峰值达到 25。
7. 使用扮演身份卡锚定角色边界。
8. 完成至少 1 次扮演反思。
9. 完成至少 1 次雾都报社轮班。

## 阶段目标

| 里程碑 | 目标时间 |
|---|---:|
| 抵达营地 | 00:10:00 |
| 晋升序列 9 | 00:30:00 |
| 晋升序列 8 | 01:00:00 |
| 晋升序列 7 | 01:30:00 |
| 锚定身份 | 01:40:00 |
| 完成反思 | 01:50:00 |
| 完成城市谋生 | 02:00:00 |

`report` 还记录首次神秘生物击杀、首次有效扮演和首次风险峰值，便于定位材料、战斗或恢复节奏卡点。

## 连续性证据

- 退出并重新连接。
- 专用服务器停止并重新启动。
- 主世界与其他维度往返。
- 死亡后重生并恢复 Capability。

试炼状态、身份锚、反思计数、城市轮班和里程碑使用当前 schema 19 保存；数据会在 Clone、重连、维度切换和服务器重启后恢复。

## 自动验证

```bash
python scripts/check_m1_playability.py
./gradlew test
./gradlew runGameTestServer
python scripts/run_server_restart_matrix.py --timeout 180
```

通过标准：核心目标 9/9、连续性 4/4、七个阶段均有记录；任何崩溃、复制、永久卡步、跨玩家串档或客户端越权均为阻断项。
