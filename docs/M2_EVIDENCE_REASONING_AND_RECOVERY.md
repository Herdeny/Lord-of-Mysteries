# M2 证据关联、案件分析与恢复

> 适用版本：`0.9.13-1.20.1` · Minecraft 1.20.1 · 网络协议 12 · Capability schema 19

本批把“已经收集了哪些证据”扩展为可执行的调查循环：玩家可以看到证据之间如何相互印证、形成矛盾或指向下一条线索；系统同时给出当前推论、档案完整度和不会剧透未取得证据的下一步提示。关键案件物品遗失时，可以在真实调查板附近安全恢复，避免长流程软锁。

## 最短使用流程

1. 在雾都前哨使用调查板接取案件。
2. 调查板第一页查看案件与当前步骤，第二页查看原始证据，第三页查看关联推理。
3. 不方便打开界面时输入 `/pm case analyze` 查看同一份服务端分析。
4. 输入 `/pm case hypothesis` 查看已揭示关系 ID，并在调查板附近提出、检验或复议自定义假说。
5. 输入 `/pm case sites` 重新获取调查地点方向；输入 `/pm case archive` 查看三案结案进度与评级。
6. 委托书或必要密封卷宗遗失时，站在真实调查板六格范围内输入 `/pm case recover`。

## 三阶段推理

| 阶段 | 进入条件 | 玩家获得的信息 | 不会发生的事 |
|---|---|---|---|
| 收集证据 | 已记录证据不足一半 | 当前缺口与最直接的调查入口 | 不提前揭示未取得证据或卷宗真伪 |
| 关联核验 | 至少一半证据已记录，但关键判断条件未完成 | 当前路线、已建立的关联和下一目标 | 不替玩家自动完成步骤或提交判断 |
| 可以判断 | 案件关键证据条件成立 | 当前推论、完整关联链和结案方向 | 不绕过 NPC、地点、战斗、护送或结算 |

“档案完整度”按服务端已经确认或标记为矛盾的证据占全部证据的比例计算。矛盾线索是有效发现，不等于无效证据；它通常用于证明配方、证词或时间线存在问题。

## 关系类型

| 类型 | 含义 | 示例 |
|---|---|---|
| 相互印证 | 两条已取得记录描述同一地点、身份或时间线 | 幸存记者证词与夜袭记录 |
| 形成矛盾 | 已取得证据和正规标准不一致 | 水印、墨迹或材料顺序出现异常 |
| 指向下一线索 | 当前记录给出下一调查位置或许可入口 | 离站记录指向最后营地 |

缺少任一端证据的关系会保持“未取得”，不会因为客户端打开界面而自动解锁。已经揭示的关系会同时提供稳定 ID，供玩家假说引用；服务端仍以关系真实类型进行检验。

## 玩家假说与误判恢复

- `/pm case hypothesis` 显示当前草稿、状态、0–3 点未解决推理负担，以及当前已经揭示的关系 ID。
- 玩家可用 `supports`、`contradicts` 或 `leads_to` 立场绑定一条关系和最多 160 字说明；修改必须靠近真实调查板。
- 错误检验增加 4 点压力和 1 点推理负担；检验冷却 10 秒。正确检验可恢复 1 点旧负担。
- 距上次检验 30 秒后，可在调查板复议并恢复 1 点负担；`clear` 只清草稿，不抹除错误责任。
- 每点结案时仍未解决的负担扣 4 点程序分，最多扣 12 点；基础奖励完全不变。

完整命令、关系 ID、迁移和防滥用规则见 [M2 玩家假说与误判恢复](M2_PLAYER_HYPOTHESES_AND_REASONING_STRAIN.md)。

## 三案推理链

### 走失的煤球

| 证据链 | 作用 |
|---|---|
| 报社最后目击记录 → 营地足迹 | 把宽泛的失踪范围收束到调查员营地 |
| 营地足迹 ↔ 煤球发现位置 | 核对目标确实沿同一路线移动，并确认可安全交还 |

下一步会依次引导报社核对、前往营地、寻找煤球和返回联络员，不会直接生成煤球或跳过接近目标的步骤。

### 失踪调查小队

| 证据链 | 作用 |
|---|---|
| 调查队离站记录 → 最后营地 | 建立失踪前的时间和路线连续性 |
| 营地遗留物 ↔ 染血笔记 | 确认笔记属于同一支调查队 |
| 染血笔记 → 调查许可 | 让联络员根据教堂与仪式信息开放后续行动 |
| 笔记记录 ↔ 幸存记者证词 | 交叉核验记者身份、邪教营地与撤离路线 |
| 幸存者证词 ↔ 夜袭记录 | 核验袭击时间并形成可提交的完整报告 |

路线选择、护送协调者、夜袭波次和多人离线追赶仍使用原有服务端权威状态；推理页只解释这些状态，不另建客户端进度。

### 真假配方

| 证据链 | 作用 |
|---|---|
| 神秘学家小屋登记 ↔ 密封卷宗 | 建立卷宗来源和保管链 |
| 鉴定规范 ↔ 水印/墨迹/材料顺序 | 分别检查纸张、书写材料和配方结构 |
| 三项证据 → 综合判断 | 只有完成三项检查后才形成一致或伪造矛盾推论 |

卷宗隐藏真伪只保存在服务端物品 NBT。客户端收到的是鉴定后的只读结论键；错误判断仍增加失控压力，推理页不会代替玩家提交 verdict。

## 命令

| 命令 | 作用 | 距离要求 |
|---|---|---|
| `/pm case` | 保持原有调查地点导航 | 无 |
| `/pm case sites` | 显式重新显示营地、教堂、邪教营地和小屋方向 | 无 |
| `/pm case analyze` | 聊天中显示完整度、阶段、推论、下一步和已建立关系 | 无 |
| `/pm case hypothesis` | 显示假说、负担与已揭示关系 ID | 无 |
| `/pm case hypothesis propose <立场> <关系ID> <说明>` | 提出支持、矛盾或指向假说 | 必须在真实调查板六格范围内 |
| `/pm case hypothesis test\|reconsider\|clear` | 检验、恢复一次负担，或仅清除草稿 | 必须在真实调查板六格范围内 |
| `/pm case archive` | 显示三个正式案件的结案数量、评级与平均分 | 无 |
| `/pm case debrief [案件]` | 显示摘要或指定案件的四维评分、耗时、路线与建议 | 无 |
| `/pm case recover` | 恢复当前活动案件缺失的关键物品 | 必须在真实调查板六格范围内 |

## 安全恢复规则

- 所有活动案件都可以恢复遗失的委托书；背包、副手或末影箱中已有委托书时不会重复发放。
- 真假配方只在步骤 3–5 需要卷宗时恢复密封卷宗；领取前和完成判断后的阶段不会额外生成。
- 新卷宗使用同一世界种子、玩家 UUID 和接案时间重新计算，因此隐藏真伪与原卷宗一致，不能通过反复恢复刷新答案。
- 在步骤 5“提交判断”恢复的卷宗保留已鉴定状态；步骤 3–4 恢复的卷宗仍须完成正常鉴定，因此不会卡步或跳步。
- 染血调查笔记继续通过最后营地现场恢复；煤球、记者和调查 NPC 继续由既有幂等生成与协调逻辑维护。
- 恢复命令不推进任务、不发放报酬、不修改证据状态，也不能在远离调查板时调用。

## 服务端与网络边界

- `CaseEvidenceView` 在服务端根据活动委托、任务步骤、知识状态和卷宗证据生成。
- `CaseAnalysisStage` 只包含 `NO_CASE / COLLECTING / CORRELATING / READY` 四个稳定枚举值。
- `EvidenceRelationKind` 只包含 `SUPPORTS / CONTRADICTS / LEADS_TO` 三个稳定枚举值。
- 0.9.7 调查板 S2C 快照加入完整度、统计、阶段、推论键、下一步键和关系列表，协议从 10 升到 11；0.9.9 再加入稳定关系 ID 与假说工作区，协议升至 12。
- 消息数量仍为 14，既有 discriminator 0–13 未重排；旧协议客户端必须被严格拒绝，不能静默误读扩展快照。
- 恢复资格、调查板邻近、物品存在性、卷宗阶段、假说关系可见性和冷却全部由服务端重新检查。

## 当前边界

- 当前关系由三个已发布案件的服务端规则生成；玩家可提交绑定既有关系的自定义说明，但尚未开放自由拖线或创建新证据边。
- 结案档案已保存每案最近一次耗时、路线、错误判断影响与四维评分；完整规则见 [M2 结案评价与复盘](M2_CASE_DEBRIEF_AND_RATINGS.md)。
