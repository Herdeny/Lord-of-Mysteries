# M2 持久化队伍调查与离线追赶

> 适用版本：0.9.17-1.20.1
>
> 状态：v0.9 M2 可玩迁移资产；正式队伍 GUI 与真实多人负载验收仍待完成

## 目标

原有 `coop.shared_progress` 只能同步在线、同队且处于同一任务链的玩家。0.8.8 在不增加客户端协议包、
不改变 Capability schema 的前提下，加入世界级队伍调查账本，使已经参加案件的成员离线后仍能追赶，
同时保留每名玩家独立领取报酬、声望和冷却的规则。

## 组队规则

1. 使用原版 `/team` 建立记分板队伍；任务定义必须启用 `shared_progress=true`。
2. 队伍完整成员表必须为 2–4 人，离线成员同样计入 `max_party=4`。
3. 每名玩家自行接取案件，或执行 `/pm party sync` 明确加入现有队伍调查。
4. 初始 5 人以上队伍安全降级为个人进度，不创建部分共享状态；已经登记的四人账本在队伍后来扩容时继续服务原成员，但拒绝新成员加入。
5. 不同队伍使用不同持久键；记者使用每队独立护送实例，夜袭追兵和救援路线也不会跨队串档。
6. 离开或切换记分板队伍时，旧账本成员关系自动移除；个人任务副本保留并继续按单人规则运行，进行中的记者护送会派生个人实例。

## 队伍调查账本

主世界 SavedData `lord_of_mysteries_quest_parties` 按 `team:<队伍名>` 保存：

- 委托与任务链稳定 ID。
- 当前步骤、目标计数和接取时间。
- 记者 UUID、夜袭生成状态和下一波时间。
- 强攻 / 潜入 / 占卜路线及准备状态。
- 已登记成员和已经独立结算的成员。
- 最近一次真实状态变化的世界 tick。

账本只接受同一任务链的更远步骤，落后的离线副本不能把队伍进度回退。活动委托冲突时也不会覆盖玩家
已有案件。最后一名登记成员结算或退出后，完成账本会从世界存档删除。

## 玩家命令

| 命令 | 用途 |
|---|---|
| `/pm party` | 查看队伍名、案件、步骤、目标计数、在线待结算人数、登记人数和账本更新时间 |
| `/pm party sync` | 明确加入当前队伍案件，或立即追赶到持久账本的最新步骤 |
| `/pm commission status` | 查看同步后的个人案件目标与救援路线 |
| `/pm commission abandon` | 个人退出队伍案件，不会清除其他成员的进度 |

已登记成员重新登录时会自动恢复，不要求手动执行 `sync`。首次中途加入的成员必须满足案件前置条件，
且不能已有另一份活动委托。缺失的调查委托书、当前玩家绑定的真假配方卷宗、动态证物袋和已取得样本会在恢复后自动补发。背包满时这些任务物不会掉落到世界形成副本；清出空位后可重试恢复。

动态轮换案件额外要求所有成员使用同一 `commissionAcceptedTick`。新成员接取或加入已有账本时，服务端先把个人接案 tick、步骤和共享状态对齐到权威账本，再进入在线参与者集合；因此即使成员跨游戏日加入，也会获得与队长相同的案件实例、现场角色和封存样本，而不会用新一天重新生成真相。

## 个人结算

- 任务步骤共享，报酬、组织声望、完成记录和冷却按玩家分别结算。
- 一名成员先结算后会被标记为已结算，登录时不会被重新加入或重复领取奖励。
- 已完成不可重复委托的玩家不能通过 `/pm party sync` 再次加入；未来可重复委托仍必须通过冷却。
- 尚未结算的离线成员会保留完成步骤，上线后可以正常返回对应 NPC 或委托板结算。
- 放弃只移除当前成员；其余成员仍可继续。所有成员均离开后账本自动清理。

## 恢复矩阵

| 场景 | 预期 |
|---|---|
| 队员离线、队友推进 | 离线成员上线后自动采用更远步骤和目标计数 |
| 队友完成整条任务 | 离线成员恢复到待结算状态，仍需个人领取奖励 |
| 协调者离线 | 在线 UUID 最小成员接管护送/夜袭，不重复生成整场波次 |
| 服务端重启 | 世界 SavedData、每队记者实例、夜袭、路线和成员结算状态共同恢复 |
| 玩家死亡或跨维度 | Capability 克隆保留个人副本，下一次对账只前进不回退 |
| 已有其他案件 | 拒绝同步，不覆盖个人活动委托 |
| 动态案件跨日加入 | 先采用队伍账本权威接案 tick，再补发当前实例证物袋与已取得的封存样本 |
| 接案前队伍超过四人 | 禁止创建共享账本，继续使用安全的个人任务状态 |
| 已登记后队伍扩容 | 原成员继续使用既有账本，第五名及以后成员不能加入 |
| 成员已结算 | 不重新下发案件，不重复发放货币或声望 |
| 成员换队或退队 | 从旧账本删除成员关系，保留个人任务状态；护送记者派生个人实例，不影响旧队 |
| 结算时队伍已超员 | 按 UUID 与案件 ID 注销，不依赖当前队伍规模 |
| 损坏账本 | 非法键、UUID、完成账本和越界步骤安全清理或拒绝 |

## 自动门禁

- `QuestPartySnapshotTest` 与 `QuestPartyPolicyTest` 覆盖 NBT/SavedData 往返、最远进度合并、冲突保护、人数上限、已登记成员扩容续玩、已结算成员和落后副本防回退。
- M2 调查合同检查持久存储名、关键 NBT 字段、登录/退出事件、结算钩子、每队护送实例、任务物无掉落交付、命令和中英双语键。
- Forge 专用服务器冒烟继续验证数据加载、命令注册、保存和干净停服。
- 当前自动化基线为完整 Gradle 测试、v0.9 内容图与统一 JSON/双语/模型/注册资源门禁；真实 2–4 人掉线、
  管理员离线改队、队名复用和长时间运行仍需人工矩阵。

## 当前边界

- 这是基于记分板队伍的可靠命令反馈，不是最终客户端队伍 GUI。
- 新成员需要主动执行 `/pm party sync`，系统不会在未确认时强制覆盖其委托。
- 账本不跨服务器迁移，也不替代世界备份。
- 真实多人带宽/tick 预算、队名复用管理、换队界面、队长权限和邀请流程仍属于后续 M2 alpha 工作；本版已保证扩容、拆队、离线追赶和独立护送不会直接锁死现有流程。
