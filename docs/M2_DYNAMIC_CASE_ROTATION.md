# M2 八槽位动态案件轮换

> 适用版本：`0.9.13-1.20.1`
> 设计基线：Project Mystery v0.9
> 存档：Capability schema 19 · 内容 schema v4
> 网络：协议 12 · 14 个固定消息 ID
> 最后更新：2026-07-24 03:24:54 UTC+01:00

## 结论

M2 现已提供第一版**可重复游玩的动态案件闭环**。服务器每天按世界种子与接案时间生成一份稳定案件，轮换覆盖失踪者、异常物品和神秘犯罪三类模板；每七日还会在侦探事务所、雾都报社与巡警系统间轮换协调组织。接案会发放绑定当前实例的实体证物袋，并在真实生成地点呈现具备关系与四段日程的案件相关人、受影响者和可直接采样的可视物证。玩家完成现场采样后会获得黄铜筹码、墨迹、烛蜡或残缺时刻表之一，再继续 NPC 证词和报纸档案核验；也可使用命令或调查板替代路线，随后识别植入式误导、提交三选一结论，并在误判后完成可恢复复议。结案评级还会对本周协调组织产生可恢复的声望反馈。

这不是“完整城市案件模拟器”的完成声明。当前版本冻结生成、案件角色与物证实体化、周轮值、主体日程、关系节点、组织反馈、替代入口、误判、恢复、结算和重启一致性合同；正式报社/事务所/警局街区、更深层关系演化、组织反应任务、扩展周级案件池和长期经济仍属于后续 M2 工作。

## 解锁与开始

1. 完成 `走失的煤球`、`失踪的调查小队·第一阶段` 和 `真假配方`。
2. 前往雾都调查板；右键调查板会把 `雾都轮换调查` 作为下一项推荐委托。
3. 也可以先执行 `/pm case rotation` 查看今日预览；预览不会显示真相槽位。
4. 接案后会自动获得“轮换案件证物袋”；再次执行 `/pm case rotation` 可查看案件编号、类型、案件周、协调组织、主体关系、日程与八个生成槽位，异常、动机、手段和掩饰会按现场、证词、档案与复议逐步揭示。
5. 到指定现场寻找带名称的案件相关人、受影响者与悬浮物证；手持证物袋右键悬浮物证或附近方块封存样本，再右键指定证人和当日报纸，完成三轮实体取证；随后提交结论并回调查板结算。

委托基础奖励为 **60 便士**和侦探事务所 **4 点声望**，冷却为 **24 游戏小时**；S/A/B/C/D 评级还会对本周协调组织追加 `+3/+2/+1/0/-2` 声望。同一个世界在同一天的预览稳定；案件接取后使用接案 tick 固定实例，因此跨午夜、重连和服务器重启都不会改变正在调查的真相、组织、关系或日程。

## 生成合同

案件类型不是八槽位的一部分，而是模板外壳。类型按游戏日稳定轮换：

- `missing_person`：失踪者案件；实地证人默认是报社办事员。
- `anomalous_item`：异常物品案件；实地证人默认是神秘学鉴定师。
- `occult_crime`：神秘犯罪案件；实地证人默认是值夜者联络员。

每份案件包含以下八个槽位：

| 槽位 | 当前候选数 | 作用 |
|---|---:|---|
| 主体 `subject` | 4 | 决定受调查的人或社会身份 |
| 动机 `motive` | 4 | 提供证词可以验证的行为原因 |
| 手段 `method` | 6 | 与最终原因类别严格配对 |
| 地点 `location` | 4 | 指向现有可到达的调查结构 |
| 异常 `anomaly` | 4 | 构成第一条客观真线索 |
| 掩饰 `cover_up` | 4 | 构成自然误导和植入式伪证 |
| 受害影响 `victim_impact` | 4 | 描述案件对家庭、街区或秩序的反馈 |
| 关键物证 `evidence_theme` | 4 | 为后续现场物品化和美术扩展提供稳定语义 |

生成器使用常数时间的确定性混合，不调用客户端随机数。`world seed + case day` 决定今日预览，`world seed + commissionAcceptedTick` 决定已接案件；实例编号同时包含类型、游戏日和短哈希，便于截图、Issue 和多人复盘对齐。

## 周级轮值、关系与日程

案件周按 `floor(gameDay / 7)` 计算。世界种子和案件周会在三个现有组织间稳定轮换，避免服务端重启或成员重连导致协调方漂移：

| 协调组织 | 声望存档键 | 当前作用 |
|---|---|---|
| 侦探事务所 | `detective_agency` | 调查委托与基础结算组织 |
| 雾都报社 | `mist_city_press` | 报纸、档案和新闻反馈组织 |
| 巡警系统 | `constabulary` | 安全室、秩序与巡查反馈组织 |

主体槽位同时确定一条关系和一段日程：

| 主体 | 关系节点 | 日程 | 观察窗口 |
|---|---|---|---|
| 记者 | 编辑主管 | 报社晨班 | 早晨 |
| 账房 | 受其供养的亲属 | 码头午后交接 | 下午 |
| 草药铺学徒 | 店铺导师 | 药铺晚班 | 傍晚 |
| 退休巡警 | 前巡逻搭档 | 夜间巡查 | 夜间 |

Minecraft 一日被划分为四个各 **6000 tick** 的时间段。活动现场每 100 tick 重新核对：窗口开放时主体移动到观察位置，关闭时返回日常位置；`/pm case rotation` 与主体交互会显示当前时间段和下一窗口等待分钟数。观察窗口只提供现场语境，**不是任务硬门禁**；证物直接采样、`field` 和 `desk` 路线在任何时段都可继续。

## 证物袋

“轮换案件证物袋”是当前案件的可见调查凭证，但不是权威存档本体：

- NBT 绑定案件实例 ID、案件类型、关键物证和 0–3 归档阶段，堆叠上限为 1。
- 悬浮提示显示案件编号、关键物证和“等待现场 / 现场已封存 / 证词已归档 / 全部归档”；阶段 3 时产生附魔光效。
- 每次队伍推进会刷新所有在线参与者持有的当前实例证物袋；中途 `/pm party sync` 或离线追赶成功时自动发放当前实例和阶段，单人不能用修改 NBT 代替服务端目标记录。
- 遗失时站在真实调查板附近执行 `/pm case recover`，只补回当前实例并恢复服务端记录的阶段；已有证物袋不会重复发放。
- 正确结算或主动放弃会回收当前实例证物袋；残留旧实例因案件 ID 不匹配不能用于后续轮换。

## 现场实体化与差异化样本

站点只有在对应结构真实生成并加载后才会创建动态现场，避免在预估坐标、未生成地形或错误高度留下实体。每个活动实例包含三类现场表现：

| 表现 | 生成规则 | 玩家用途 | 生命周期 |
|---|---|---|---|
| 案件相关人 | 按 `subject` 槽位生成记者、账房、草药铺学徒或退休巡警职业外观，并按日程在日常点/观察点移动 | 右键查看主体身份、关系、当前窗口与卷宗指引；不替代正式证人口供 | 活动实例存在时按需重建，旧实例在站点加载后清理 |
| 受影响者 | 按 `victim_impact` 生成焦急家属、缺货商贩、受盘查居民或谣言目击者 | 右键确认案件的家庭、街区、秩序或传闻后果，以及本周协调组织 | 与案件实例绑定，不提供额外任务进度 |
| 可视物证 | 以不可拾取的持久展示实体显示当前 `evidence_theme` | 手持当前证物袋直接右键采样；也可右键 28 格内附近方块 | 无活动在线实例时清理，返回后由服务端重建 |

四类封存样本都是独立注册物品，使用原版纹理作为当前可见占位美术：

| 关键物证槽位 | 封存物品 | 资源 ID |
|---|---|---|
| 黄铜筹码 | 封存黄铜筹码 | `lord_of_mysteries:dynamic_case_brass_token` |
| 墨迹 | 封存墨迹样本 | `lord_of_mysteries:dynamic_case_ink_trace` |
| 烛蜡 | 封存烛蜡样本 | `lord_of_mysteries:dynamic_case_candle_wax` |
| 残缺时刻表 | 封存残缺时刻表 | `lord_of_mysteries:dynamic_case_torn_schedule` |

样本 NBT 同时绑定案件实例 ID 与物证主题。现场步骤完成后，所有在线参与者各获得一份个人封存镜像；迟到成员、离线追赶成员和遗失物品的成员按服务端步骤补发，已有样本不会重复生成。结算或放弃时，证物袋与本轮封存样本一起归档。展示实体本身不可拾取，修改或复制物品 NBT 也不能替代服务端目标记录。

## 三轮取证与多入口

每条必需证据都有一个优先的直接世界交互、一个兼容实地命令和一个调查板替代入口。世界交互零费用并让线索可见可操作；`field` 保留对旧流程和无障碍操作的兼容，`desk` 保证 NPC 死亡、结构未加载、报纸遗失或成员离线时不会软锁。

| 阶段 | 优先世界交互 | 兼容 `field` | 调查板 `desk` | 记录内容 |
|---|---|---|---|---|
| 现场 | 携带当前证物袋，右键案件可视物证或指定地点 28 格内的现场方块 | 玩家位于指定地点 28 格内 | 复原现场记录并补发封存样本 | 异常槽位 + 差异化样本 |
| 证人 | 右键当前类型指定的标签 NPC | 玩家位于指定 NPC 10 格内 | 复原离线口供 | 动机槽位 |
| 档案 | 右键当日报纸 | 背包或任一手持有报纸 | 调取备份 | 手段槽位 |

命令：

```text
/pm case rotation investigate field
/pm case rotation investigate desk
```

调查板复原优先收取 **6 便士**档案费。余额不足时不会卡死任务，而是改用高负荷记忆回溯并增加 **3 点失控压力**。费用或压力只作用于发起操作的玩家；任务步骤通过既有 2–4 人持久队伍账本同步。若使用 `desk` 完成现场步骤，也会发放当前主题的封存样本，保证直接路线与恢复路线的物品状态一致。

直接世界交互和 `field` 最终调用同一服务端推进路径，因此距离、步骤、案件实例、NPC 标签、报纸持有、队伍同步和重复提交具有相同校验；客户端物品动画或伪造 NBT 不会单独产生进度。

## 证据板与因果关系

轮换案件复用调查板的证据页、关联推理页、玩家假说和结案复盘，不新增客户端自报状态。

固定显示五条证据记录，并在关联推理页组成七条关系：

1. 案件简报：接案后立即确认。
2. 现场真线索：记录生成的异常。
3. 证词真线索：记录生成的动机。
4. 档案真线索：记录与结论一致的手段。
5. 植入式误导：记录生成的掩饰，初始状态为 `SUSPICIOUS`。

固定关系 ID：

| 关系 ID | 类型 | 何时揭示 |
|---|---|---|
| `brief_to_scene` | `LEADS_TO` | 取得现场证据 |
| `schedule_to_scene` | `LEADS_TO` | 取得现场证据 |
| `scene_to_testimony` | `SUPPORTS` | 取得证词 |
| `relationship_to_testimony` | `SUPPORTS` | 取得证词 |
| `testimony_to_records` | `LEADS_TO` | 取得档案 |
| `false_lead_conflict` | `CONTRADICTS` | 取得档案；误判复议后转为确认冲突 |
| `synthesis_to_conclusion` | `SUPPORTS` | 三轮取证完成 |

这些稳定 ID 可以直接用于玩家假说：

```text
/pm case hypothesis propose contradicts false_lead_conflict <你的说明>
/pm case hypothesis test
```

每日案件复用同一委托 ID，但接取新实例时会清除上一实例的假说草稿，避免旧案件说明污染新案件；历史结案档案保留该委托最近一次评分。

## 结论、误判与恢复

三个可提交结论：

```text
/pm case rotation conclude human_concealment
/pm case rotation conclude extraordinary_distortion
/pm case rotation conclude ritual_diversion
```

手段与真相的服务端映射：

| 结论 | 可生成手段 |
|---|---|
| 人为掩藏 `human_concealment` | 伪造离开、假交接转移 |
| 非凡扭曲 `extraordinary_distortion` | 记忆覆盖、镜面替身 |
| 仪式诱导 `ritual_diversion` | 仪式引诱、象征置换 |

正确结论会完成任务链，锁定复盘路线，并提示回调查板结算。错误结论不会清空案件或销毁证据，而会：

- 增加 **6 点失控压力**；
- 把共享案件状态切换为 `reconsider`；
- 禁止继续穷举结论，直到完成恢复节点；
- 保留已经收集的三类证据。

恢复流程：

```text
/pm case rotation recover
# 或沿用统一入口
/pm case recover
```

恢复必须在真实调查板附近执行。成功后状态切换为 `recovered`，恢复 2 点压力，把植入式误导标记为已经确认的矛盾，然后允许再次提交结论。恢复状态、正确结论和接案 tick 都进入既有队伍 SavedData，因此在线队员与重连成员看到同一案件状态。

## 结算与组织反馈

动态案件继续先执行原有委托结算：**60 便士**、侦探事务所 **4 点基础声望**和 24 游戏小时冷却。随后按照同一份服务端权威复盘评级，对本周协调组织追加一次反馈：

| 评级 | 组织声望变化 | 反馈 |
|---|---:|---|
| S | +3 | 显著认可 |
| A | +2 | 正向认可 |
| B | +1 | 谨慎认可 |
| C | 0 | 中性记录 |
| D | -2 | 可恢复的负面评价 |

反馈复用玩家现有 `orgReputation` 映射并标记 Social dirty 区，不新增存档字段。它只在成功结算时执行一次；活动委托随后清除，因此重复点击、旧证物袋或客户端自报评级都不能重复改写声望。

## 服务器权威边界

- 客户端只发送既有调查板意图或执行 Brigadier 命令；案件生成、距离、NPC、报纸、余额、压力、结论与恢复都由服务端校验。
- 未新增网络消息；协议保持 `12`，消息总数保持 `14`。
- 未新增玩家字段；Capability schema 保持 `19`。
- 委托和任务链继续使用 schema v4 JSON，并由内容图、双语、跨文件引用和至少双解法门禁校验。
- 组队成员共享步骤、接案 tick、结论与恢复状态；玩家假说、金钱和风险仍保持个人责任边界。
- 新成员加入已有队伍账本时，先采用账本的权威接案 tick，再进入参与者同步；即使跨越游戏日，也不会在同队生成两个不同轮换实例。
- 证物袋与封存样本只是服务端状态的可见镜像；恢复、队伍阶段同步、结算和放弃回收均由服务端执行。
- 案件相关人、受影响者和可视物证使用原版实体加服务端持久数据，不新增玩家存档字段；旧实例实体不能推进当前案件，已加载现场会清理不再活动的实例。
- `field` 与调查板替代入口确保 NPC 死亡、离线成员、物品遗失与结构暂未加载不会形成不可逆软锁。
