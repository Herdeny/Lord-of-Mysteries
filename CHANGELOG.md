本项目采用 [Semantic Versioning](https://semver.org/) 的预发布阶段规则，并在版本号后附加
Minecraft 兼容版本。版本与更新时间以 `project-status.json` 为准。

## [0.9.10-1.20.1] - 2026-07-19

### Added

- 新增每日可重复的“雾都轮换调查”：按世界种子和接案 tick 稳定生成失踪者、异常物品、神秘犯罪三类案件，以及主体、动机、手段、地点、异常、掩饰、受害影响、关键物证八个槽位。
- 新增现场、证人、档案三轮双入口取证；实地入口连接现有调查地点/NPC/报纸，调查板替代入口在资源缺失时以 6 便士或 3 点压力保证无软锁推进。
- 新增人为掩藏、非凡扭曲、仪式诱导三类结论和植入式误导；错误结论增加 6 点压力并进入 `reconsider`，调查板复议后标记伪证冲突、恢复 2 点压力并允许重试。
- 新增五条轮换案件证据关系、玩家假说兼容、24 游戏小时冷却、60 便士/4 声望结算、最近一次轮换复盘，以及完整中英文本和玩家/开发者手册。
- 新增动态生成确定性、三日类型轮换、手段—结论一致性、证据阶段、伪证复议、队伍恢复状态、存档清洗和复盘路线测试；JUnit 基线增至 248 项。

### Changed

- M2 数据合同扩展为 4 个委托、4 条任务链、八槽位生成、双入口、三结论与恢复状态；内容图增至 116 节点、163 关系。
- 调查板、案件分析、玩家假说、持久队伍和结案复盘复用同一轮换案件状态；接取新实例时清理上一实例假说草稿，避免跨日污染。
- 专服冒烟不再硬编码委托/任务链数量，而是从主资源目录计算预期值，后续新增内容不会制造假失败。
- README、ROADMAP、Pages、Wiki、入门指南、专服矩阵、M2 计划与实现基线同步到 0.9.10。

### Compatibility

- 未新增 Capability 字段，schema 保持 19；未新增网络包，协议保持 12，14 个固定消息 ID 不变。
- 正在进行的轮换案件使用接案 tick 固定真相并通过既有队伍 SavedData 保存，因此跨午夜、重连和服务器重启不会重新生成答案。

### Security

- 地点距离、NPC 存活/标签、报纸、余额、压力、步骤、结论、恢复和结算全部由服务端验证；客户端不能提交证据、真相、奖励或复盘分数。
- NPC 死亡、物品遗失、结构暂未加载和离线队员都有调查板恢复入口；错误结论不能靠连续穷举跳过复议节点。

## [0.9.9-1.20.1] - 2026-07-18

### Added

- 新增按案件持久化的玩家假说工作区：可把最多 160 字的自定义说明绑定到已经揭示的稳定关系 ID，并选择支持、矛盾或指向立场。
- 新增 `/pm case hypothesis`、`propose`、`test`、`reconsider` 与 `clear` 命令，以及调查板“检验假说 / 复议”按钮和只读快照展示。
- 新增可恢复误判代价：错误检验增加 4 点失控压力与最多 3 点推理负担；正确检验或调查板复议可逐点恢复。
- 新增假说消毒、NBT 往返、协议快照、错误/纠正/复议冷却、结案扣分与 schema 18→19 迁移测试，JUnit 基线增至 239 项。

### Changed

- 证据关系快照现在携带稳定关系 ID；隐藏关系仍不会发送给客户端，也不能通过猜测 ID 提前建立假说。
- 每点未解决推理负担在结案时扣 4 点程序分，最多扣 12 点；该评价不改变便士、声望、物品或知识奖励。
- 玩家 Capability 升至 schema 19；命名 DataFix `case_hypothesis_workspace` 为旧档建立空工作区并保留非法载荷到孤儿数据。
- 网络协议升至 12，14 个固定消息 ID 保持原顺序；README、ROADMAP、Pages、Wiki 与 M2 手册同步到 0.9.9。
- 更新 Mod 内置简介到当前 M2 玩法，并强制 `processResources` 使用 UTF-8，修复中文 `mods.toml` 在发布 JAR 中被写成 GBK 的乱码问题。

### Security

- 所有假说修改都要求真实调查板邻近并由服务端重验活动案件、已揭示关系、冷却和有界文本；客户端不能提交证据、分数或奖励变化。

## [0.9.8-1.20.1] - 2026-07-18

### Added

- 三个正式案件结算时自动生成证据 40、程序 30、安全 20、效率 10 的四维 100 分复盘，并按 S/A/B/C/D 五档评级。
- `/pm case debrief` 显示结案档案摘要，三个固定案件子命令可查看详细分数、耗时、路线和改进建议；`/pm case archive` 同步显示评级与平均分。
- Capability 新增 `case_debriefs` 持久归档，保存每案最近一次复盘；schema 17 旧档通过命名 DataFix 自动升级并保留历史完成状态。
- 新增结案评分、边界钳制、NBT 往返与 schema 17→18 迁移测试，JUnit 基线增至 229 项。

### Changed

- 玩家 Capability 升至 schema 18；网络协议保持 11，14 个固定消息 ID 不变。
- 真假配方结算先快照三项鉴定证据与错误判断次数，再回收卷宗，避免复盘丢失程序数据。
- README、ROADMAP、Pages、Wiki、入门指南、调查板手册、健康基线与 M2 合同同步到 0.9.8。

### Security

- 评分只在服务端确认案件可结算后生成；客户端不能提交分数、评级、路线或耗时。
- 复盘不修改便士、声望、物品或知识奖励；损坏归档保留为孤儿数据，不阻断其余玩家存档。

## [0.9.7-1.20.1] - 2026-07-18

### Added

- 调查板新增第三页“关联推理”：显示档案完整度、确认/矛盾/缺失统计、收集—关联—判断三阶段、当前推论与服务端给出的明确下一步。

- 新增案件恢复策略测试、推理阶段/关系/可信度测试和扩展协议往返测试，JUnit 基线增至 224 项。

### Changed

- 调查板快照扩展推理摘要和关系列表，网络协议升级为 11；14 个既有消息 ID 保持不变且未重排。
- M2 自动合同现在同时保护推理阶段、关系类型、可信度、下一步、结案档案、恢复命令、调查板距离门禁和协议稳定性。
- README、ROADMAP、Pages、Wiki、玩家指南、专服矩阵和开发基线同步到 0.9.7。

### Security

- 推论、证据关系、卷宗真伪、恢复资格和物品发放均由服务端计算；未发现关系不会进入客户端快照，客户端不能伪造关联、远程恢复或复制关键物品。
- 恢复的真假配方卷宗保持原始确定性真伪，并按任务阶段保留已完成鉴定，避免重掷答案、跳步或在提交判断前永久卡案。
