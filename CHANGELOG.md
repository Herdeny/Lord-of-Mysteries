# Changelog

本项目采用 [Semantic Versioning](https://semver.org/) 的预发布阶段规则，并在版本号后附加
Minecraft 兼容版本。版本与更新时间以 `project-status.json` 为准。

## [0.9.3-1.20.1] - 2026-07-17

### Added

- `docs/master/m0_runtime_catalog.json` 与多目录内容图构建，将实现证据扩至 114 节点、160 关系；生成配方增至 13 份。
- `restore_migration_backup.py` 与隔离回滚检查，支持 dry-run、manifest 校验、恢复前安全快照、精确替换、快照后文件清除和路径穿越拒绝。
- 双启动专服矩阵，连续验证相同世界种子、schema 17 marker、定义加载、诊断、保存与干净停服。
- 扮演身份卡、反思日志和首次服药身份套件；`/pm recover` 可补回身份物品。
- 雾都报社生活班次与 `/pm life`，用纸张换取便士和面包并进入 M1 验收。
- 带精确 `CharacteristicBundle` 的破碎非凡特性；永久失控时转移到实体并在死亡时守恒掉落。
- 两小时 M1 合同：9 项核心目标、7 个时间里程碑，以及身份、反思和城市生活持久化状态。
- 2 项 Forge GameTest，覆盖精确特性载荷和两小时试炼往返；真实 GameTest 总数增至 5 项。

### Changed

- 玩家 Capability 由 schema 16 升至 17，旧档自动回填身份、反思、城市生活与两小时试炼字段。
- 新手雾都前哨扩展床、工作区、街道和生活补给桶；调查员生存纵切从 60 分钟升级为 120 分钟。
- Build CI 增加回滚演练并将单次专服冒烟替换为双启动矩阵。
- `project-status.json` 与 `roadmap.json` 将 M0/M1 标记为完成，正式里程碑推进到 M2；README、Pages、Wiki 和验收文档同步更新。

### Fixed

- Windows GBK 终端输出不再导致专服重启矩阵在服务器已成功时误报编码失败。
- 可恢复崩溃与永久失控不再混用无载荷特性碎片，避免特性凭空复制或丢失。

### Added

- 精确导入 v0.9 完整设计、增量说明和 manifest；`scripts/import_v09_design.py` 校验完整文档与内嵌 v0.8 基线的哈希、字节数、行数，并接入 Gradle 和 GitHub Actions。

- `docs/master/m0_content_catalog.json` 建立首批 M0 物品、状态、配方与知识门槛迁移目录；Build 与 Documentation Consistency 均执行内容图门禁。
- 分层 `CharacteristicBundle` / `CharacteristicLedger`，保存途径、序列、纯度、印记、污染与来源哈希；五条途径的每次魔药晋升写入特性账本。
- 扮演 v2 基础字段：原则理解、角色过度认同、反思次数与最近反思日；新增每日 `/pm reflect`，并在状态包和非凡者档案显示。
- 自动化测试增至 191 项，覆盖 schema 16 迁移、分层特性、扮演身份与 schema v4 内容元数据。
- v0.9 差异报告、实施基线、M0 内容图/迁移说明、代码审计表及对应 Wiki 页面。

### Changed

- 路线从 v0.8 M0–M7 重构为 v0.9 M0–M12；当前里程碑重置为 M0，既有五途径、委托、任务链和雾都内容统一标注为迁移资产。
- 玩家 Capability schema 从 15 升至 16；旧 schema 15 非凡者存档按当前途径和序列自动回填 legacy 特性层。
- SimpleChannel 协议从 6 升至 7，以同步原则理解和角色过度认同。
- 永久失控清除当前途径身份但保留特性账本，为后续提取、掉落实体和守恒闭环保留可信数据。
- README、ROADMAP、Pages、Wiki、版本规范与 CI 文案全面切换到 v0.9 事实口径，区分设计规格与仓库实现证据。

### Fixed

- Capability Provider 现在在 Forge 生命周期结束时正确失效，避免无效对象继续持有玩家数据。
- 生成器、运行时加载器和图谱校验使用同一组 schema v4 来源与引用约束，减少文档、生成资源和运行时定义漂移。
- 专用服务器烟测不再硬编码网络协议号，而是从 `NetworkProtocol.PROTOCOL_VERSION` 读取预期值，避免协议升级后错误超时。

## [0.8.9-1.20.1] - 2026-07-17

### Added

- 统一资源完整性门禁 `scripts/check_resource_integrity.py`，一次检查 253 个 JSON、760 个中英双语键、330 个静态引用键、73 个模型以及 68 物品、5 方块、8 实体的注册资源。

- `/pm servercheck` 新增 `active_parties` 与 `party_members` 运行指标，专服烟测会验证账本可访问且运行指标可读。

### Fixed

- 玩家离开或切换原版记分板队伍后，旧调查账本现在会删除其残留成员关系；放弃与结算不再依赖当前队伍人数或当前队伍名。
- 已完成不可重复委托的玩家无法通过 `/pm party sync` 再次加入并重复领取奖励；可重复委托同步同样遵守冷却。

- 任务计数使用长整型安全累加，避免损坏进度与大额增量发生整数回绕。

### Changed

- 委托与任务链运行时解析现在和生成器执行同一组硬校验：非空/唯一字段、0–9 等级、合法失败策略、2–4 人共享上限、正数目标计数、自依赖与冷却溢出保护。
- Forge 入口改用 47.4.20 支持的构造上下文注入，Gradle 发布仓库改为赋值语法，清除项目自身对两个将来会移除的加载 API 与旧 Maven DSL 的使用；`downloadMcpConfig` / `reobfJar` 仍会报告 ForgeGradle 6 内部弃用提示。
- Build 与 Documentation Consistency 工作流均执行统一资源门禁；Capability schema 保持 15，网络协议保持 6。
