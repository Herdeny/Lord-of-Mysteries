四种委托和四条任务链继续可玩；占卜家纵切归入已完成 M1，其余横向途径按 M3 门禁继续扩展。

分层非凡特性和扮演 v2 身份区分；0.9.1 补齐正式 DataFix、迁移前世界快照、
原始 NBT 备份和未知/非法 ID 孤儿保留，0.9.2 完成四区 dirty mask 与生命周期摘要同步，
0.9.3 完成受限回滚、双启动专服矩阵与 M1 两小时玩法闭环；0.9.4 新增服务端权威调查板 UI；0.9.5 加入三案证据档案和每日动态报纸；0.9.6 为新旧前哨补齐事务所、巡警功能亭与首批城市服务；0.9.7 增加证据关联、三阶段推理、明确下一步和关键物品恢复；0.9.8 增加持久化结案评分与复盘；0.9.9 加入玩家自定义假说、误判代价和可恢复推理负担；0.9.10 加入三类八槽位动态案件、双入口取证、植入式误导和误判复议；0.9.11 把三轮取证落为证物袋现场采样、NPC 证词和报纸档案交互；0.9.12 在真实站点生成案件相关人、受影响者和可直接采样的可视物证，并发放四类实例绑定封存样本；0.9.13 增加七日组织轮值、四段主体日程、关系证据节点与结案组织反馈。网络协议保持 12，当前正式里程碑为 M2。

1. 首次登录取得调查手稿与罗盘；使用 `/pm next` 查看当前主线，遗失时用 `/pm recover` 自助补回。
2. 右键罗盘或输入 `/pm camp` 前往世界种子确定的新手营地，取得专用补给桶和随机线索箱。
3. 用补给桶中的坩埚、灵性草药、占卜水晶和月华水炼制并服用序列 9 占卜家魔药。
4. 通过占卜、克制与帮助他人完成序列 9 消化，再炼制序列 8 小丑魔药。
5. 使用纸牌飞刃、完美平衡、表情操控与直觉闪避消化小丑魔药。
6. 携带灵视提灯狩猎幻形蛇，取得毒腺后炼制并服用序列 7 魔术师魔药。
7. 用 `/pm status`、`/pm m1check` 和阶段自动提示核对晋升、能力、灵性、风险与下一门槛。
8. 每个游戏日可使用 `/pm reflect` 反思一次，降低角色过度认同和压力并提高原则理解。
9. 验收时使用 `/pm trial start`；`/pm trial report` 输出营地和三个序列的阶段耗时，`verify` 核对恢复门禁。
10. 完成占卜家纵切后输入 `/pm mistcity` 前往雾都前哨；右键调查板比较案件并明确接取。
11. 接案后依次查看“证据档案”和“关联推理”，检查档案完整度、证据关系、当前推论与下一步；聊天模式可用 `/pm case analyze`。
12. 在调查板附近用 `/pm case hypothesis` 查看已揭示关系 ID；提出 `supports`、`contradicts` 或 `leads_to` 假说并检验。误判会增加 4 点压力与推理负担，纠正成功或等待后“复议”可逐步恢复。
13. 完成失踪小队后接取“真假配方”，用 `/pm case` 前往神秘学家小屋；鉴定三项证据后再提交 authentic/forged 结论。
14. 潜行交互报社职员完成每日轮班并获得报纸；右键报纸阅读当日城市头条、案件提示和轮班状态。
15. 输入 `/pm city` 查看服务目录：普通交互事务所打开调查板，潜行交互购买补给；巡警岗亭可查看风险，潜行交互申请安全室照护。
16. 委托书或真假配方必要卷宗遗失时，站在调查板附近输入 `/pm case recover`；已有物品不会重复发放。
17. 结案前处理未解决推理负担；结案后用 `/pm case archive` 查看平均分，或用 `/pm case debrief lost_cat|missing_squad|counterfeit_formula|dynamic_case` 重看详细复盘。
18. 2–4 人联机时使用原版 `/team` 组队；`/pm party` 查看持久账本，离线成员上线自动追赶，中途加入者用 `/pm party sync`。
19. 完成“真假配方”后接取“雾都轮换调查”；到指定现场辨认案件相关人、受影响者和悬浮物证，携带证物袋右键物证或附近方块封存差异化样本，再右键指定证人与当日报纸完成三轮取证。`/pm case rotation` 会显示本周协调组织、主体关系、当前日程窗口及等待时间；错过观察窗口不会锁死证据。`investigate field|desk` 仍是兼容/恢复入口；误判后在调查板执行 `recover`。

### v0.9.13 M2 周轮值、角色日程与组织反馈

- 世界种子和案件周稳定轮换侦探事务所、雾都报社与巡警系统三个协调组织；同一世界每七个游戏日切换一次，活动案件在跨日、重连和重启后保持接案时的组织与真相。
- 记者、账房、草药铺学徒和退休巡警分别采用早晨、下午、傍晚和夜间日程；每段持续 6000 tick，主体会在日常位置与观察位置之间移动。
- 观察窗口提供额外现场语境而非硬性任务门禁；错过时 `/pm case rotation` 会给出下一窗口等待时间，证物、`field` 和 `desk` 三条取证路线始终保留。
- 四类主体关系进入证词语义，证据板新增 `schedule_to_scene` 与 `relationship_to_testimony` 两个稳定关系 ID；每份动态案件现在显示七条关系。
- 结案评级会对本周协调组织追加声望反馈：S/A/B/C/D 分别为 `+3/+2/+1/0/-2`。反馈复用现有组织声望存档，不新增 Capability 字段，负面结果可由后续案件恢复。
- Capability schema 保持 19，网络协议保持 12/14 包；自动化基线现为 268 项 JUnit、6 项 Forge GameTest、116 节点/168 关系内容图，以及 273 个 JSON、1190 对双语键、466 个静态引用键、80 个模型和 75 个物品资源。
- 完整轮值、日程、关系、恢复边界与验收清单见 [`docs/M2_DYNAMIC_CASE_ROTATION.md`](docs/M2_DYNAMIC_CASE_ROTATION.md)。

### v0.9.12 M2 案件主体与差异化物证

- 每个已加载的真实案件地点会按活动实例生成一名案件相关人、一名受影响者和一个可视物证；三者都绑定服务端实例 ID，旧现场不能推进新卷宗。
- 主体身份随记者、账房、草药铺学徒或退休巡警槽位变化；受影响者随家属恐慌、街区短缺、巡警严查或神秘传闻变化，右键可查看案件背景与当前卷宗指引。
- 可视物证按黄铜筹码、墨迹、烛蜡和残缺时刻表四类显示；手持证物袋可直接右键展示物采样，也保留右键附近现场方块的无障碍入口。
- 现场完成后每名在线队员会取得绑定案件实例与物证主题的封存样本；中途加入、离线追赶和 `/pm case recover` 会按服务端步骤补齐，但不会重复发放。
- 结案或放弃会同时回收证物袋与封存样本；无人继续调查时，已加载旧现场的主体、受影响者和展示物会自动清理，需要时可由权威状态重建。
- 新成员接入已有队伍账本时先采用权威接案 tick，避免跨游戏日加入导致同队生成不同案件；Capability schema 19 与网络协议 12/14 包保持不变。
- 自动化基线现为 260 项 JUnit、6 项 Forge GameTest、116 节点/168 关系内容图，以及 273 个 JSON、1159 对双语键、80 个模型和 75 个物品资源。
- 完整交互、恢复边界和验收清单见 [`docs/M2_DYNAMIC_CASE_ROTATION.md`](docs/M2_DYNAMIC_CASE_ROTATION.md)。

### v0.9.11 M2 实体证物与世界交互

- 接取每日轮换案件会获得绑定实例 ID、案件类型、关键物证与归档阶段的“轮换案件证物袋”；旧案件证物袋不能推进新实例。
- 现场阶段携带证物袋前往卷宗指定结构，在 28 格范围内右键任一现场方块即可封存异常样本。
- 证人阶段直接右键卷宗指定的报社办事员、神秘学鉴定师或值夜者联络员；档案阶段直接右键当日报纸。
- 每次推进都会刷新队伍在线成员的证物袋阶段；三项证据齐全后物品发光，方便确认已可提交结论。
- 证物袋遗失时在真实调查板附近执行 `/pm case recover`，系统按服务端步骤恢复当前阶段；结案或放弃会自动回收。
- `field` 命令和收费 6 便士/压力 +3 的 `desk` 路线完整保留，NPC 死亡、物品丢失和结构暂未加载仍不会造成软锁。
- Capability schema 保持 19，网络协议保持 12/14 包；自动化基线增至 251 项 JUnit、6 项 Forge GameTest、116 节点/164 关系内容图。
- 完整交互、恢复边界和验收清单见 [`docs/M2_DYNAMIC_CASE_ROTATION.md`](docs/M2_DYNAMIC_CASE_ROTATION.md)。

### v0.9.10 M2 八槽位动态案件轮换

- 每个游戏日稳定轮换失踪者、异常物品和神秘犯罪三类案件；世界种子与接案 tick 固定案件编号和真相，跨午夜、重连与重启不漂移。
- 每案生成主体、动机、手段、地点、异常、掩饰、受害影响和关键物证八个槽位；手段与人为掩藏、非凡扭曲、仪式诱导三种结论保持严格因果对应。
- 现场、证人、档案三轮必需证据各有实地入口与调查板替代入口。替代入口收取 6 便士；余额不足时增加 3 点压力但不会软锁。
- 证据板新增五条案件记录和 `brief_to_scene` 等五个稳定关系；植入式误导以 `CONTRADICTS` 关系进入玩家假说与服务端推理链。
- 错误结论增加 6 点压力并进入 `reconsider`，禁止连续穷举；在真实调查板执行 `/pm case rotation recover` 后识别伪证、恢复 2 点压力并重新结论。
- 轮换委托支持 2–4 人持久队伍，共享接案 tick、步骤、结论和恢复状态；费用、压力与玩家假说保持个人责任。
- 奖励为 60 便士与 4 点事务所声望，冷却 24 游戏小时；新实例会清除上一轮假说草稿，结案档案保存最近一次轮换评分。
- Capability schema 保持 19，网络协议保持 12/14 包；自动化基线增至 248 项 JUnit、6 项 Forge GameTest、116 节点/163 关系内容图。
- 完整生成合同、命令、关系 ID、多人边界和验收清单见 [`docs/M2_DYNAMIC_CASE_ROTATION.md`](docs/M2_DYNAMIC_CASE_ROTATION.md)。

### v0.9.9 M2 玩家假说与误判恢复

- 每条已揭示证据关系现在都有稳定 ID；玩家可把最多 160 字的自定义说明绑定到该关系，并选择“支持 / 矛盾 / 指向”立场。
- `/pm case hypothesis` 列出当前可用关系；`propose`、`test`、`reconsider`、`clear` 修改操作必须在真实调查板附近完成，未揭示关系无法被猜 ID 提前引用。
- 错误检验增加 4 点失控压力和 1 点未解决推理负担，负担最多 3 点；检验冷却 10 秒，调查板复议冷却 30 秒，防止刷取和无代价试错。
- 用正确立场重新检验可恢复 1 点负担；调查板复议也可清除草稿并恢复 1 点。单纯 `clear` 只清草稿，不抹除既有错误代价。
- 结案时每点未解决推理负担扣 4 点程序分，最多扣 12 点；结案前纠正或复议即可恢复，因此错误有反馈但不会永久软锁。
- 三案结算时自动生成证据 40、程序 30、安全 20、效率 10 的四维评分，并按 90/80/65/50 阈值给出 S–D 评级。
- 复盘记录总分、四项分数、接案到结案耗时、强攻/潜入/占卜路线与下一次唯一改进建议；评分不改变原有报酬。
- `/pm case archive` 汇总完成度、每案评级和已评级平均分；`/pm case debrief <案件>` 可随时重看详细复盘。
- 真假配方在回收卷宗前快照三项证据与错误判断次数；每次错误判断扣 5 点程序分，最多扣 15 点。
- Capability 升至 schema 19；schema 18 通过 `case_hypothesis_workspace` 自动增加空假说工作区，旧结案与既有玩家状态不被改写。
- 网络协议升至 12，14 个固定消息 ID 不重排；自动化基线增至 239 项 JUnit、6 项 Forge GameTest、1014 对双语键和 424 个静态引用。
- 完整规则、关系 ID 和恢复流程见 [`docs/M2_PLAYER_HYPOTHESES_AND_REASONING_STRAIN.md`](docs/M2_PLAYER_HYPOTHESES_AND_REASONING_STRAIN.md)。

```bash
./gradlew compileJava test
./gradlew build
./gradlew runGameTestServer
./gradlew runClient
./gradlew runServer
python scripts/import_v09_design.py --check
python scripts/gen_datapack.py --check
python scripts/build_content_graph.py
python scripts/check_save_rollback.py
python scripts/run_server_restart_matrix.py --timeout 180
python scripts/check_m2_investigation.py
python scripts/check_resource_integrity.py
python scripts/check_github_text_encoding.py --repository Herdeny/Lord-of-Mysteries
```

构建产物位于 `build/libs/`，放入 Forge 1.20.1 的 `mods/` 目录即可加载。

## 版本与文档同步

`project-status.json` 是版本号、开发阶段、技术基线和最后更新时间的唯一来源。
功能变更必须同时更新 README、Pages、Wiki 和 `CHANGELOG.md`，然后运行：

```bash
python scripts/sync_project_metadata.py
python scripts/import_v09_design.py --check
python scripts/gen_datapack.py --check
python scripts/build_content_graph.py
python scripts/check_resource_integrity.py
python scripts/check_github_text_encoding.py --repository Herdeny/Lord-of-Mysteries
python scripts/sync_project_metadata.py --check
```

CI 会检查 v0.9 设计源、元数据同步、116 节点/168 关系内容图、M1 两小时合同、M2 调查/证据/推理/玩家假说/动态案件角色与差异化物证/七日组织轮值/主体日程/关系节点/组织反馈/误判恢复/结案复盘/报纸/城市服务合同、统一资源完整性、GitHub Issue/评论编码和存档回滚，运行 268 项 JUnit、6 项 Forge GameTest，并连续两次启动 Forge 专用服务器验证同一世界重启；`wiki/` 内容变化后自动发布 GitHub Wiki。完整规则见
[`VERSIONING.md`](VERSIONING.md)。

## 项目结构

```text
src/main/java/top/aurora/lordofmysteries/
├── ability/       # 灵视、危险直觉、简易占卜与可信度
├── acting/        # 扮演事件、消化公式、身份区分、反思与反刷取
├── artifact/      # 封印物行为
├── characteristic/# 分层非凡特性、来源哈希与迁移账本
├── client/        # 按键与非凡者档案 UI
├── commission/    # 委托与任务链 schema v4 运行时定义
├── command/       # /pm 自助引导与阶段检查
├── content/       # schema v4 公共来源、剧透、关系与实现元数据
├── entity/        # 失控体与非凡生物
├── knowledge/     # 调查手稿与阶段引导
├── network/       # 服务端权威网络包
├── player/        # Capability、灵性、污染与失控
├── potion/        # 坩埚、魔药品质与晋升
├── ritual/        # 多方块阵列、仪式状态机、结算与失败风险
├── registry/      # 方块、物品、方块实体和创造栏注册
└── world/         # 营地生成、坐标持久化与新手入口
```

数据定义位于 `src/main/resources/data/lord_of_mysteries/`，静态资料站位于 `docs/`，
发布地址为 <https://herdeny.github.io/Lord-of-Mysteries/>。
