# v0.9 实施基线

当前版本 `0.9.5-1.20.1` 以 v0.9 为设计基线，M0/M1 实施完成，正式里程碑进入 **M2 调查与生活**。仓库现有行为优先于设计示例，设计规格数量不等于实现数量。

## 本批完成

- 精确导入完整设计、增量说明和 manifest，并在 CI 校验哈希、字节和行数。
- 公共内容 schema v4 元数据进入途径、15 个现有序列、3 个委托和 3 条任务链。
- 内容图当前包含 114 节点、160 关系，并生成孤儿、剧透、兼容、本地化和资产报告。
- 玩家 Capability 升至 schema 17，旧 schema 15/16 存档自动迁移并回填 M1 状态。
- 五条现有途径的序列晋升记录分层特性、纯度、印记、污染与来源哈希。
- 扮演 v2 新增身份卡、反思日志、原则理解、角色过度认同和每日反思。
- 状态界面同步新扮演字段；网络协议升至 10，并新增服务端权威调查/证据快照。
- 首次部署在世界加载前创建 schema 17 快照，并用正式 DataFix、精确恢复、原始 NBT 备份和 `orphaned_entries` 无损连接旧 M1/M2 存档。
- Core/Knowledge/Social/Endgame 四区 dirty mask、生命周期核心摘要、客户端缓存和 6 项 Forge GameTest 已进入 Build CI。
- 两小时 M1 合同、9 项核心目标、7 个里程碑、永久失控特性精确掉落和城市报社工作闭环完成。
- 三案证据档案与确定性每日城市报纸完成，报社轮班连接情报产出。

## 当前边界

- M0 与 M1 的机器可验证门禁已经关闭；真人两小时、死亡/跨维度和多人长时记录持续回归。
- 特性重复合并和更高序列完整守恒仍属于后续里程碑。
- JEI、EMI、Jade 目前只有只读兼容计划。
- 五途径序列 9–7、三委托和三任务链继续作为迁移资产；M2/M3 仍需独立验收。
