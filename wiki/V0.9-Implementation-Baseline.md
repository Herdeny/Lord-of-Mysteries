# v0.9 实施基线

当前版本 `0.9.2-1.20.1` 以 v0.9 为设计基线，正式里程碑保持 **M0 内容基建**。仓库现有行为优先于设计示例，设计规格数量不等于实现数量。

## 本批完成

- 精确导入完整设计、增量说明和 manifest，并在 CI 校验哈希、字节和行数。
- 公共内容 schema v4 元数据进入途径、15 个现有序列、3 个委托和 3 条任务链。
- 内容图当前包含 71 节点、63 关系，并生成孤儿、剧透、兼容、本地化和资产报告。
- 玩家 Capability 升至 schema 16，旧 schema 15 非凡者自动回填特性 bundle。
- 五条现有途径的序列晋升记录分层特性、纯度、印记、污染与来源哈希。
- 扮演 v2 新增原则理解、角色过度认同和每日 `/pm reflect` 反思。
- 状态界面同步新扮演字段，网络协议升至 8。
- 首次部署在世界加载前创建 schema 16 快照，并用正式 DataFix、原始 NBT 备份和 `orphaned_entries` 无损连接旧 M1/M2 存档。
- Core/Knowledge/Social/Endgame 四区 dirty mask、生命周期核心摘要、客户端缓存和 3 项 Forge GameTest 已进入 Build CI。

## 当前边界

- 71 节点尚未达到 M0 的 100 节点门禁。
- 首批 Capability Clone/迁移 GameTest 已完成；自动重启/回滚和真人死亡/跨维度整体验收尚未完成。
- 特性提取、实体掉落、合并和完整死亡守恒未完成。
- 两小时教程、身份卡、反思日志物品和城市生活闭环未完成。
- JEI、EMI、Jade 目前只有只读兼容计划。
- 五途径序列 9–7、三委托和三任务链是迁移资产，不代表 M1–M3 完成。
