# v0.9 实施基线

当前版本 `0.9.18-1.20.1` 以 v0.9 为设计基线，M0/M1 已完成，正式里程碑仍是 **M2 调查与生活**。M2 自动化范围已经闭环但四人八小时真人门禁未完成；M3 五途径序列 6–5 已开始 `code_ready` 实施。仓库现有行为优先于设计示例。

## 本批完成

- 精确导入完整设计、增量说明和 manifest，并在 CI 校验哈希、字节和行数。
- 公共内容 schema v4 元数据进入途径、25 个现有序列、4 个委托和 4 条任务链。
- 内容图当前包含 126 节点、188 关系，并生成孤儿、剧透、兼容、本地化和资产报告。
- 玩家 Capability 升至 schema 23，旧 schema 15–22 存档自动迁移并回填 M1/M2 状态、三职业班次与神秘暴露。
- 五条现有途径的序列晋升记录分层特性、纯度、印记、污染与来源哈希。
- 扮演 v2 新增身份卡、反思日志、原则理解、角色过度认同和每日反思。
- 状态界面同步新扮演字段；网络协议升至 12，并新增服务端权威调查/证据/推理/假说快照。
- 首次部署在世界加载前创建 schema 23 快照，并用正式 DataFix、精确恢复、原始 NBT 备份和 `orphaned_entries` 无损连接旧 M1/M2 存档。
- Core/Knowledge/Social/Endgame 四区 dirty mask、生命周期核心摘要、客户端缓存和 9 项 Forge GameTest 已进入 Build CI。
- 两小时 M1 合同、9 项核心目标、7 个里程碑、永久失控特性精确掉落和城市报社工作闭环完成。
- 固定三案、三类八槽位动态案件、案件相关人/受影响者/可视物证、四类封存样本、七日组织轮值、主体关系/日程、组织评级反馈、19 条证据关系、三阶段推理、明确下一步、误判复议、关键案件物品恢复与确定性每日城市报纸完成，报社轮班连接情报产出。
- 0.9.14 完成同站点 25 槽实例隔离、展示物实例校验、主世界昼夜统一、玩家绑定配方卷宗、队伍专属记者、满背包任务物防掉落，以及已登记队伍扩容续玩。
- 0.9.15 增加每周组织指令、八案有界结案历史、旧待处理后续自动过期，以及按 S–D 评级发放的组织回访、声望、金钱与压力恢复；领取后续必须无活动委托并站在真实调查板附近。
- 0.9.16 增加四名联系人独立态度与两阶段实体组织响应；正确组织 NPC 简报、调查板提交、三日过期和主动放弃均由服务端校验。
- 0.9.17 增加最多 24 条联系人事件和优先/常规/关系修复分支；旧历史回填记忆，旧活动任务无损迁移。
- 0.9.18 生成三座 7×7 正式功能街区，旧世界迁移到 `service_version=2`；动态案件扩为 6 类关系、12 项周指令和五档组织立场。
- 0.9.18 加入三职业共享日限经济、0–100 神秘暴露、S–D 暴露变化和 16 日六事件周期。
- 0.9.18 开始 M3 五途径序列 6–5：10 瓶魔药、20 项服务端能力、20 项扮演事件和五途径 7→6→5 重启 GameTest。
- 三案结案评价、玩家假说、动态案件历史、联系人/组织响应、城市经济与暴露通过 schema 23 持久化。

## 当前边界

- M0 与 M1 的机器可验证门禁已经关闭；真人两小时、死亡/跨维度和多人长时记录持续回归。
- 特性重复合并和更高序列完整守恒仍属于后续里程碑。
- JEI、EMI、Jade 目前只有只读兼容计划。
- 五途径序列 9–7、四委托和四任务链继续作为可玩迁移资产。
- M2 保留四人八小时真人专服与经济平衡门禁；M3 序列 6–5 仅为 `code_ready`，专属晋升仪式、完整系统和组队互补未完成。
