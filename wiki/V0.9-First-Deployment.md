# v0.9 首次实施部署

> 当前版本：`0.9.1-1.20.1` · 首次部署先保护旧存档，不先改玩法数值
>
> 历史说明：本页保留 0.9.1 首次迁移流程。当前 0.9.16 使用 Capability schema 21；恢复前请先运行仓库中的隔离回滚检查与恢复工具。

## 已接入的保护

- 世界加载前在 `project_mystery_backups/schema-16-<UTC 时间>/` 创建迁移快照。
- 快照覆盖 `level.dat*`、玩家 `.dat` 和 Project Mystery SavedData，使用临时目录与原子替换。
- `schema-16.complete` 保证同一 schema 只创建一次快照；备份失败会阻止世界继续加载。
- schema 0/15→16 使用命名 DataFix 链，原始 Capability NBT 进入 `migration_backups`。
- 非法或未来 ID、损坏 bundle 和不能安全解释的旧字段进入 `orphaned_entries`，不再静默删除。
- M1 试炼、M2 委托/调查、余额、队伍账本、途径、序列、冷却和扮演状态继续沿用。

## 首次启动检查

1. 停服后额外复制完整世界目录。
2. 替换 JAR，启动并确认日志出现 schema 16 备份成功信息。
3. 用旧玩家检查 `/pm status`、`/pm trial status`、`/pm party`、`/pm servercheck`。
4. 正常停服并重启，确认状态一致且没有重复快照。

回滚时必须停服、换回旧 JAR，并从同一快照整体恢复 `level.dat*`、`playerdata/` 和 Project Mystery SavedData，不能混合新旧文件。

完整步骤与验证范围见[主仓库部署手册](https://github.com/Herdeny/Lord-of-Mysteries/blob/main/docs/V0.9_FIRST_DEPLOYMENT.md)。
