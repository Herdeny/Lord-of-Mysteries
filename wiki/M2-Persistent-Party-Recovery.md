# M2 持久化队伍调查与离线追赶

> 当前版本：0.9.17-1.20.1 · Capability schema 22 · 网络协议 12

0.8.8 为 `shared_progress=true` 调查加入主世界持久化队伍账本。已登记成员离线时，在线队友仍可推进；
成员重新登录后会自动追赶到队伍最新步骤，同时保留每名玩家独立结算报酬、声望和冷却。

## 快速使用

1. 用原版 `/team` 建立 2–4 人记分板队伍。
2. 各成员接取同一案件；中途加入者执行 `/pm party sync`。
3. 用 `/pm party` 查看案件、步骤、目标计数、在线人数、待结算人数和更新时间。
4. 队员离线后继续推进；其上线时会自动收到恢复提示。
5. 每名成员分别返回 NPC 或委托板结算，避免共享奖励重复发放。

## 安全边界

- 首次登记时离线成员计入 `max_party=4`；初始 5 人以上安全降级为个人进度。
- 已登记的 1–4 人账本在记分板队伍后来扩容时继续可玩；原成员仍按持久名单同步，新加入的第 5 人不会进入旧账本。
- 账本只接受更远步骤，落后副本不能回退队伍状态。
- 已有另一份活动委托时拒绝同步。
- 动态轮换案件的中途成员先采用账本权威接案 tick；跨游戏日加入也会取得与队友相同的案件实例、现场角色和封存样本。
- 已结算成员不会被重新加入或重复领取奖励。
- 放弃只移除当前成员；最后一名成员退出或全员结算后自动清理账本。
- 换队或退队会自动清理旧账本成员关系；个人任务状态不会被删除，并在记者护送中为新个人/队伍创建独立记者。
- 已完成不可重复委托不能通过 `/pm party sync` 重复领取奖励。
- 只有已登记且未结算成员可以合并进度；损坏键、UUID、完成账本和越界步骤会安全清理。
- 不同队伍的记者、夜袭、救援路线和任务步骤相互隔离；旧存档若仍指向营地公共记者，会在加载后自动迁移为队伍专属护送记者。
- 委托书、配方卷宗、轮换证物袋和封存样本在背包满时不会掉到地面；清出空间后可在对应 NPC 或调查板恢复，避免复制和公共拾取。

## 保存内容

`lord_of_mysteries_quest_parties` 保存委托/任务 ID、步骤、计数、接取时间、记者 UUID、夜袭状态、
三解路线、登记成员、已结算成员和更新时间。该账本最初随 Capability schema 17/网络协议 10 引入；当前玩家 Capability schema 为 19，网络协议为 12。

## 恢复结果

| 场景 | 结果 |
|---|---|
| 离线期间队友推进 | 上线自动追赶 |
| 队友已完成任务 | 恢复为个人待结算状态 |
| 协调者离线 | 在线 UUID 最小成员接管 |
| 服务端重启 | 账本、路线、记者和夜袭状态恢复 |
| 死亡/跨维度 | 个人 Capability 保留，并与账本只前进对账 |
| 冲突案件/首次超员队伍 | 拒绝覆盖或安全降级 |
| 已登记后队伍扩容 | 原 1–4 人继续共享，第 5 人保持独立且不能污染账本 |
| 换队/退队 | 清理旧账本关系，保留个人任务副本；护送记者按新队伍分叉 |
| 背包满/任务物遗失 | 不向世界掉落；清出空间后按当前玩家、案件和接取时间恢复 |
| 超员后个人结算 | 仍按 UUID 与案件 ID 正确注销 |
